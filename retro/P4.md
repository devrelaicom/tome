# Phase 4 Retrospective: User Story 2 — Author a catalog that the tool accepts

## What Worked Well

- **The Phase 2 validator was already correct.** Every rejection case in
  `tests/path_validation.rs` and every `ManifestInvalid` variant in
  `tests/manifest_strictness.rs` passed first try — no source changes were
  needed. The corpus is a regression net, not new behaviour.
- **Table-driven negative tests with `tempfile::TempDir`.** Each row stands on
  its own and reads as documentation: build a catalog root, write a manifest,
  call `parse_and_validate`, assert the precise `ManifestInvalid` variant.
- **`std::os::unix::fs::symlink` for the escape test.** Phase 1 only targets
  Unix (macOS + Linux per the constitution), so the symlink-escape test can
  use the unconditional Unix-only API. Documented at the top of
  `path_validation.rs`.

## What Didn't Work

- **`format!(r#"…"#)` with no substitutions** triggers `clippy::useless_format`
  on the `-D warnings` clippy profile. Replaced with raw string literals.
  Easy fix, but a reminder that pasted-from-spec test snippets should use
  plain `&str` unless they actually interpolate.

## Workarounds & Solutions

- The `unknown_owner_field_rejected` test uses `[owner.extra]` instead of
  adding a field on the existing `[owner]` table because TOML's `[owner]`
  appearing twice would be a duplicate-table error, not an unknown-field
  error. Subtle, worth noting if anyone extends the corpus.

## Packages & Dependencies

- No new crates. `toml`, `tempfile`, `tome` (lib) are sufficient.

## Patterns & Code

- **One test per error variant.** `tests/error_messages.rs` is small and
  mechanical: instantiate each variant with placeholder fields, call
  `to_string()`, assert that the result contains the field path, value, and
  key. If anyone tweaks an `#[error("…")]` template in `src/error.rs`, this
  test catches the regression.
- **Two test layers complement each other.** `manifest_strictness.rs` covers
  parse-time strictness via the actual `parse_and_validate` path;
  `error_messages.rs` covers the rendered string. A regression in either
  layer surfaces independently.

## For Next Time

- Phase 5 audits the contributor on-ramp documentation. The retro template
  mentions a 10-minute target (SC-002); on a fresh clone, with cargo's
  network access and a cold cache, the test suite alone takes ~30 seconds.
  The 10-minute budget should hold comfortably.
- A future "manifest version bump" — if `tome-catalog.toml` adds new fields —
  needs to extend both the strictness corpus and the path-validation corpus.
  Make this a checklist item in the change-log entry for that bump.
