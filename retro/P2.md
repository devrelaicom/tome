# Phase 2 Retrospective: Foundational Modules

## What Worked Well

- **Capability-organised modules.** Splitting `catalog/` into `git`, `manifest`,
  `store` (plus `mod`) made the file boundaries align with responsibilities.
  Each file owns one concern and has its own unit tests.
- **`thiserror` + a closed enum.** The compiler-enforced exhaustive match in
  `tests/exit_codes.rs` is exactly the safety net the constitution asked for —
  adding a variant breaks the test file, which is the point.
- **`time::serde::rfc3339`.** Cleaner than `chrono` for our needs and saves a
  crate. The RFC 3339 wire format is unchanged.
- **`tempfile::NamedTempFile::persist`.** Same-directory temp + rename is one
  function call and the docs are unambiguous about atomicity on the same FS.
- **Lefthook + cogito + typos integration.** Caught one real typo (`mis-detected`
  in design docs) before the first commit, which paid for the install.

## What Didn't Work

- **First credential-scrubber design under-covered "Authorization: Bearer X".**
  The naive regex matched `Authorization: Bearer` as a single key+value pair and
  left the actual token (`X`) trailing. Surfaced immediately by the table-driven
  test. Documented inline; design now consumes an optional second keyword token
  in the value position.
- **SHA1-in-prose vs SHA1-after-`:`/`=`.** The original regex's lead-character
  class `[^:=A-Za-z0-9]` allowed leading spaces, so `commit: <sha>` got scrubbed
  too. Switched to a true alternation with a closure-replacer that decides per
  match. Lost the static `&[u8]` replacement convenience; gained correctness.
- **`tracing-subscriber::try_init`** silently fails if a subscriber is already
  installed. That's the right behaviour for the binary (idempotent init from
  `main`) but means tests that observe logging output need their own subscribers.

## Workarounds & Solutions

- The `reset_cancellation_for_tests` helper is `#[cfg(test)] #[allow(dead_code)]`
  until Phase 3 uses it. Removing the `#[allow]` is on the Phase 3 TODO.

## Packages & Dependencies

- Added `serde_json` (~250 KB) for the `--json` envelope and NDJSON writer. Not
  in STACK.md originally; recorded here so the codebase doc refresh picks it up.

## Patterns & Code

- **Structural grep test for `deny_unknown_fields`.** R-7's compile-time
  alternative wasn't worth the macro machinery. The 50-line grep test in
  `tests/manifest_strictness.rs` runs in CI and catches the regression we cared
  about. Same pattern is reusable for other "every X must have Y" invariants.
- **Closure replacer for `regex::bytes::Regex::replace_all`.** Lets the regex
  carry alternation while the closure decides what to emit per match — useful
  anywhere "decide based on which branch matched" applies.

## For Next Time

- Phase 3 will add a real `tests/common/mod.rs` harness that builds fixture
  catalogs in `TempDir` + `git init`. The `Git` helper in `catalog/git.rs`
  should be testable against `file://` URLs without changes; if not, that's
  a sign the helper coupled too tightly to `https://` shape.
- The kv-secret regex's two-token semantic only covers "Bearer <token>". If
  future credentials use longer multi-token shapes (e.g. `OAuth realm="x"
  token="y"`), the regex needs extending. Track in `CONCERNS.md` when the
  codebase doc refresh happens.
