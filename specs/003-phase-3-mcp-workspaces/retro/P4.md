# Phase 4 / P4 Retrospective: User Story 2 (Workspace creation + inspection)

**Phase**: User Story 2 (T104–T127)
**Spec**: [spec.md](../spec.md)
**Plan**: [plan.md](../plan.md)
**Started**: 2026-05-14
**Closed**: 2026-05-14 (US2.a + US2.b landed in one branch session)

## What Worked Well

- **Silent compute / emit wrapper pattern carried forward** (US2.a): `commands::workspace::info::assemble(&ResolvedScope, &Paths)` is the pure compute path; `run` is the thin emit wrapper. Same shape as `commands::query::pipeline` from US1.b. Tests target `assemble` directly; the CLI uses `run`. Future doctor reuse will plug straight into `assemble`.
- **Atomic-directory creation via single-FS rename** (US2.b): `tempfile::Builder::tempdir_in(workspace_root)` puts the staging dir inside the target's filesystem so `std::fs::rename` is POSIX-atomic. `TempDir::keep()` consumes the cleanup guard right before the rename — no race window. The contract specified `tempfile::TempDir::persist` (which doesn't actually exist as a `TempDir` method); the keep+rename pair gives the same atomicity guarantee.
- **Reusing the existing variant**: `CatalogAlreadyExists(String)` for workspace-already-initialised keeps the closed `TomeError` set unchanged. The exit code (4), category (`catalog_already_exists`), and JSON envelope are all that harnesses key off; the Display message is slightly off ("catalog `workspace at <path>/.tome` is already registered") but lives one level below the CLI's hint copy. The Phase 3 task plan explicitly accepted this trade-off; no specific failure mode has emerged that needs a dedicated variant.
- **`Serialize` + snake_case rename on `ScopeSource`**: gave the workspace-info JSON wire shape (`"source": "global_fallback"` etc.) for free, with one line of derive macro. The enum is already `Copy` so cloning into the record is free.
- **CLI smoke + library-API test split**: each new test file mixes library-API assertions against the pure functions (most coverage; fast; no FS round-trip through the binary) with two-or-three CLI smoke tests that prove the wiring works end-to-end. `tests/workspace_info.rs` and `tests/workspace_init.rs` both follow this shape.
- **Threaded concurrency test**: `tests/workspace_init.rs::init_concurrent_does_not_corrupt_state` uses `std::sync::Barrier::new(2)` + two threads to hit the rename-race window simultaneously. The assertion is post-condition (a single fully-formed `.tome/`, no straggler `.tome.tmp.X` dirs) rather than win-lose-counts, which would be flaky.

## What Didn't Work

- **`tempfile::TempDir::persist` doesn't exist.** The contract referenced `TempDir::persist` for atomic directory creation, but the `tempfile` 3.27 API only has `TempDir::keep()` (consume guard, return path) + a separate `std::fs::rename`. Same outcome, different shape. Noted in T116's tasks.md annotation.
- **`Config::default()` serializes as `[catalogs]\n` rather than completely empty**. The first iteration of `init_creates_dot_tome_with_empty_config` checked `!body.contains("[catalogs")` and failed because the default `BTreeMap<String, CatalogEntry>` renders an empty `[catalogs]` table in TOML. Fix: round-trip through `catalog_store::load` and assert `parsed.catalogs.is_empty()`. The on-disk representation is fine; the test was checking the wrong layer.
- **`PathBuf::from(std::env::current_dir().unwrap_or_default())`** tripped clippy's `useless_conversion` lint — `current_dir()` already returns `PathBuf`. One-line fix; flagging because the pattern crops up whenever falling back to CWD.

## Workarounds & Solutions

- **Reuse `CatalogAlreadyExists` for workspace-already-initialised**: The contract called this out explicitly. Pass a synthetic string like `format!("workspace at {}", marker.display())` so the Display message at least names the path. The cost: a slightly off-message error string. The benefit: zero new error variants, zero changes to `tests/exit_codes.rs`, zero ripple through the JSON envelope schema.
- **Best-effort cleanup of `.tome.old/`**: On `--force`, the existing `.tome/` is renamed aside to `.tome.old/` before the new directory lands. After the new `.tome/` lands successfully, the aside is `remove_dir_all`-ed best-effort. A crash mid-cleanup leaves an orphan that's harmless (doctor will surface it in a future US).
- **Pre-existing `.tome.old/` from a prior crash**: Before renaming the current `.tome/` aside, `remove_dir_all` is called on `.tome.old/` best-effort. Without this, the `rename → .tome.old` step would fail with EEXIST on the second-init-after-crash path.

## Packages & Dependencies

- **No new direct dependencies.** `tempfile = "3"` was already a dep (used by `catalog/store.rs` and `embedding/download.rs` for atomic file writes); we just exercised the directory variant for the first time.
- **`tempfile::TempDir::keep()` requires tempfile ≥ 3.13** — we're on 3.27, well above. Older versions exposed `into_path()` which is now deprecated; both work.

## Patterns & Code

- **Atomic populated-directory landing**: `Builder::tempdir_in(target_parent) → populate → keep() → std::fs::rename(staging, target)`. Distinct from `NamedTempFile::persist` (file-level atomicity). Use whenever a multi-file directory must appear either complete or not at all from a reader's perspective. Used in `src/workspace/init.rs`.
- **`--force` rename-aside-then-restore**: when replacing an existing populated directory, rename current to `.<name>.old/`, rename staging to `.<name>/`, remove `.<name>.old/` best-effort. On staging-rename failure, restore from `.<name>.old/`. The window between the two renames is the only point where the user is briefly without a complete marker.
- **Emit-only `Serialize` records**: `WorkspaceInfo`, `InitOutcome`. No `#[serde(deny_unknown_fields)]` because they're never deserialised — they exist to render JSON for stdout. Strict-on-Tome-owned-inputs still applies; this is output, not input.
- **Opt-in registry file pattern**: `inventory::append_if_registry_exists` no-ops when the file is absent. The user touches the file once to opt in; init then appends on every future invocation. Dedupe by exact-path match against the canonicalised workspace root. Same pattern is reusable for any future "let me track this for you, but only if you ask" feature.

## For Next Time

- **The "atomic directory landing" pattern wants its own helper** if a third caller emerges (US3+? doctor "auto-fix"?). The current implementation is inlined in `src/workspace/init.rs`. Rule-of-three triage applies.
- **The `.tome.old/` orphan** is currently best-effort-cleaned but not actively swept. Doctor (future US) is the natural home; for now it's silently invisible to the user. If the failure mode shows up in real usage (which it won't unless the user crashes the shell mid-init), we'll add an explicit sweep.
- **The `WorkspaceMalformed` variant**: surfaces via `info::catalog_count` when the workspace's `config.toml` won't parse as TOML. Currently surfaces only via `tome workspace info`; once `tome catalog list --workspace` etc. start running on the workspace's config (US3), the same error will surface there too. The variant name stays accurate.
- **A "workspace already initialised" dedicated variant** would clean up the slightly off Display message. Trade-off: +1 variant in the closed enum (cheap) for a cleaner error message (small UX win). Defer until a specific failure mode justifies it; for now `CatalogAlreadyExists` with a synthetic string is fine.
