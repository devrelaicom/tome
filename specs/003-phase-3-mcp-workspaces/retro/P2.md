## What Worked Well

- **Slice-per-commit cadence held**: F1–F8 each landed as one focused commit (modulo F4 which folded the deferred T021 rename's call-site sweep). Reviewer cognitive load stayed below the ~400-line PR soft cap.
- **`_for(&Scope)` accessor suffix** (F1): Adding `Paths::config_file_for(&Scope)`, `index_db_for`, `index_lock_for` next to the existing `Paths.config_file` / `index_db` / `index_lock` fields kept the working tree compiling across slice boundaries. The originally planned field rename (T021: `config_file` → `global_config_file`) turned out to be unnecessary churn — the `_for` suffix is informative enough at call sites that we shelved the rename permanently.
- **`commands::reindex::Scope` vs `crate::workspace::Scope` namespace collision** (F4): Caught at compile time inside the reindex module; fully-qualifying `crate::workspace::Scope` for the workspace one inside that module's body was a single-line fix. The collision is benign because the two types occupy different abstraction layers — reindex scope is a target enum, workspace scope is the active install context.
- **Library-API vs CLI-binary test boundary** (F6): `query::run_with_deps(args, deps, mode)` with `Embedder`/`Reranker` trait injection lets tests use `StubEmbedder` without loading the 345 MB ONNX runtime. Same pattern that landed in `commands::reindex::run_with_deps` (Phase 7) and `commands::status::assemble_report` (Phase 8). The pattern is firm now — every CLI command that wraps heavy state ships a library entry point that takes injected deps.
- **F5 read-only DB open split** (P10 deferral folded in): `open_read_only(db_path)` mirrors `open(db_path)`'s signature so existing call sites only swap the function name + drop the `OpenOptions`. Five read-side call sites updated in one commit.
- **F8's sync-boundary structural test**: Continued to pass after adding `src/mcp/` (path-component exemption matches every file under `mcp/`). The test caught zero violations during F8 — encouraging confirmation that the boundary is real.

## What Didn't Work

- **The `#[cfg(test)] thread_local!` shape in the schema-migration contract** (F7): Integration tests under `tests/` consume the library without `cfg(test)` visibility, so `MIGRATIONS_OVERRIDE` had to be unconditional. Deviated to `#[doc(hidden)] pub static` and documented the test-only intent in the doc comment. The contract should be amended in a future polish pass.
- **`SchemaTooNew` (52) vs `SchemaVersionTooNew` (73) split** (F7): The migration framework's contract names exit 73 for newer-on-disk refusal, but the F5 read-only gate (added one slice earlier) used the Phase 2 `SchemaTooNew` (52). Both gates name the same condition but route through different exit codes. Documented in both docstrings; the handover note "still surfaces exit 52" preserved the read-only behaviour, and `apply_pending` moved to the contract-mandated 73. A future polish pass could reconcile, but the split is small and intentional.
- **`tracing-subscriber`'s `MakeWriter` blanket impl for `Mutex<File>`** (F8): The `MutexGuard<File>` type doesn't impl `io::Write` directly (only `&File` does), so the obvious "use `Mutex<File>` as the `MakeWriter` source" pattern doesn't compile. Solved with a small `LockedFile<'a>(MutexGuard<'a, File>)` adapter that forwards `Write` via the `&File` blanket impl. Pattern worth remembering for any future file-backed `MakeWriter` consumer.

## Workarounds & Solutions

- **F4's `LifecycleDeps.scope: &'a Scope`**: Plumbed through every lifecycle helper (`enable_plugin_atomic`, `cascade_disable_for_catalog`, `auto_disable_orphan`, `reindex_plugin`). Tests construct `LifecycleDeps` with `scope: &tome::workspace::Scope::Global` to preserve historical behaviour.
- **F3's deliberate omission of clap `conflicts_with`**: `--workspace` + `--global` would normally use clap's `conflicts_with` attribute, but that forces exit 2 (usage error). The contract demands exit 72 (`WorkspaceConflict`). Resolution: skip `conflicts_with`, detect both-set inside `workspace::resolution::resolve`, return the dedicated error.
- **F7's `MIGRATIONS_OVERRIDE` as `#[doc(hidden)] pub static`**: Documented in this retro and the migrations.rs doc comment. The only legitimate caller is a Phase 7 integration test; production code reads `MIGRATIONS` via the `active_migrations()` helper which honours the override.
- **F8's stderr-only-on-error subscriber**: The MCP server's stdout is the protocol channel (FR-221). Logs go to a file. But fatal startup errors should still reach the harness operator without them having to find the log. Solution: a sibling `tracing-subscriber` `fmt::layer().with_filter(LevelFilter::ERROR)` on stderr alongside the JSON file layer. FR-222 honoured.

## Packages & Dependencies

- **`tracing-subscriber` `json` feature** (F8 added): Enables `fmt::layer().json()` natively. Pulls a tiny serde-json adapter; binary stayed flat at 20.94 MiB (vs 20.91 MiB baseline pre-F8). Cheaper than adding `tracing-appender` for similar functionality.
- **`tokio` minimum feature set** confirmed working (research §R-2): `["rt", "macros", "io-std", "sync", "signal", "time"]` with `default-features = false`. `rt-multi-thread` would add ~500 KB for no concurrency win on a single-client stdio server.
- **`rmcp` still unreferenced after F8**: LTO drops it from the binary entirely. The first real reference lands in US1's `src/mcp/server.rs::rmcp::ServerHandler` impl; expect the size to climb toward the ~22.8 MiB (macOS) / ~31.5 MiB (Linux) projection then.

## Patterns & Code

- **Field-method coexistence**: When a refactor would rename a public struct field across many call sites, consider adding accessor methods first (`config_file_for(&Scope)`) so the field rename can be deferred or shelved. The accessor pattern is more flexible than the field, and the suffix on the method name often makes the call site clearer than the renamed field would have.

- **Three-exit-code domain pattern** (F7 schema-migration): Where one logical condition has multiple legitimate exit codes depending on the call path, document the routing in both docstrings explicitly and lean on the closed `TomeError` set to keep the variants discoverable. F7's schema migration has 51 (`IndexIntegrityCheckFailure` — DB in unknown state), 73 (`SchemaVersionTooNew` — write-path newer-on-disk refusal), 74 (`SchemaMigrationFailed` — registered-step apply error), plus the legacy 52 (`SchemaTooNew` — read-path newer-on-disk refusal). Each variant has a distinct trigger; the four-way split is intentional.

- **Specific-over-generic exit-code preference** (F8 preflight): When a pre-flight failure matches an existing Phase 1/2 variant (`ModelMissing` 30, `ModelChecksumMismatch` 32, `EmbedderNameDrift` 41, `EmbedderVersionDrift` 42, `SchemaVersionTooNew` 73), emit that code rather than the generic residual (`McpStartupFailed` 60). The contract names the rule; the implementation just follows.

- **`LockedFile<'a>` adapter for `tracing-subscriber` `MakeWriter`** (F8): When you need a `Mutex<File>`-backed `MakeWriter`, wrap the `MutexGuard` in a thin newtype and forward `Write` via the `&File` blanket impl. Pattern works for any `Mutex<W>` where `&W: Write`.

- **Integration-test-reachable injection points via `#[doc(hidden)] pub static`** (F7): When a test injection point must be reachable from `tests/*.rs` (integration tests) AND from production code in tightly scoped diagnostic scenarios, gate with `#[doc(hidden)] pub static` rather than `#[cfg(test)]`. The integration-test crate doesn't see `cfg(test)` items from the library; `doc(hidden)` keeps the surface invisible in published docs.

## For Next Time

- **Read both contracts before splitting work between write-path and read-path gates.** F7's `SchemaVersionTooNew` (73) and F5's earlier `SchemaTooNew` (52) gate could have been reconciled if F7 had revisited F5's choice. Track the split as a polish item; it's documented but not pretty.
- **Encode pre-emptive slice splits in research §R-15** — the Phase 10 retro's recommendation. Honoured during planning; the slice plan (F1–F8 here, US1.a-d / US2.a-c / etc. later) lets each commit stay focused and the test crate keep its library-API vs CLI-binary discipline.
- **The async island remains structural-test-enforced.** No file outside `src/mcp/` reaches for `tokio`, `.await`, `tokio_*`, or `async fn`. Every PR that violates the invariant fails CI before review. Don't relax this.
- **Binary size monitoring**: Foundational closed at 20.94 MiB on macOS arm64. Research §R-2 projects ~22.8 MiB (macOS) / ~31.5 MiB (Linux) at US1 close. The 50 MB cap has headroom but the rmcp dependency tree is the load-bearing variable — measure after US1.b lands the rmcp serve loop.
