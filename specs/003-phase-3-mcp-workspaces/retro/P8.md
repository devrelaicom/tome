## What Worked Well

- **Four-reviewer parallel pass surfaced everything in one round**. Contract audit / Rust-lens code review / test audit / security audit dispatched in a single message, each writing structured findings. The disposition table (blockers → dedicated PRs, majors → grouped PRs, minors → folded, nits → wontfix-unless-trivial) is a direct lift of the Phase 2 P10 mapping; it scaled cleanly.
- **Each blocker got its own PR**. B1 (`mcp.log` field names), B2+B3 (resolver validation + doctor drift) landed separately so reviewers can revert one without dragging the others. Cheap to review, cheap to roll back.
- **Pre-emptive instrumentation (P7 → P8 lift)**. F7 had landed `MIGRATIONS_OVERRIDE` as `#[doc(hidden)] pub static`. US5 wired the e2e tests against it. Doctor's `repair_schema` was dead code until PR-E added the `subsystem: "schema"` suggested-fix emitter. The pattern of "ship the framework + the test harness, light it up later" paid off twice in one phase.
- **Custom `FormatEvent` for the contract-pinned JSON shape**. `tracing-subscriber`'s default `.json()` formatter emits `timestamp`/`message`; the contract pins `ts`/`msg`. Rather than chase a per-field-rename API, `ContractEventFormat` builds each record as a `serde_json::Map` and emits exactly what the contract names. Reusable for any future surface that needs a custom JSON shape.
- **`OnceLock`-style "first call wins" for shutdown signals**. SIGINT + SIGTERM on Unix, with `tokio::select!` returning the canonical signal literal as `&'static str`. The signal name flows directly into the log event's `signal` field; no string lookup table.
- **`5 s graceful shutdown timeout via `tokio::time::timeout(&mut waiter)`**. `RunningService::waiting()` consumes `self`, so the future is pinned once and polled from both arms of the `select!` plus the post-signal timeout. Subtle but mechanical once the lifetime ordering clicks.
- **`Orphan` as the fifth `CatalogCacheState` variant**. The classify+suggest rules became more explicit (`matches!(c.state, Missing | NotARepo | ManifestInvalid)` rather than `!= Ok`) — future variant additions are forced to think about classification.
- **Generate-at-setup fixtures over committed binaries**. PR-F's `tests/security_hardening.rs` plants registry files inline at test setup. Consistent with US5's `write_index_db_with_schema_version` pattern and Phase 2's `Fixture::build_sample` — no binary churn in PR diffs, no opaque blobs in git history.

## What Didn't Work

- **The contract had a typo (35 → 51 for `IndexIntegrityCheckFailure`)**. The variant name in `exit-codes-p3.md` is right; the number is wrong (35 is `VectorExtensionInitFailure` in the Phase 2 closed enum). Caught by the test failure when I tried to assert exit 35 and got 51. Fixed in PR-H with a note that the variant name is canonical.
- **`fixes::apply` returning `Result<usize, _>` was misleading**. Every per-fix failure was already downgraded to `warn!`; the outer `?` would have silently broken the "continue on failure" contract. PR-E changes the signature to infallible `usize` — small but load-bearing.
- **`RunningService::waiting()` consumes `self`** — can't be called twice. The original mod.rs code called it once in `select!`; trying to wait again for graceful shutdown required pinning the future first. Solved with `tokio::pin!`; would not have figured this out without the build error.
- **First-pass doctor drift test bootstrapped with stub seeds**. The DB's stored seeds disagreed with the production registry seeds → `EmbedderNameDrift` → Unhealthy even though the test was meant to assert "no drift". Fix: bootstrap with `commands::plugin::registry_seeds()` so stored == configured. Pattern: when both index seeds AND model fabricators exist in the same test, they must agree.
- **First-pass clippy hit on `1 * 1024 * 1024`**. `identity_op` denied the multiplication; changed to `1024 * 1024`. Trivial but flagged something subtle: clippy's lint set is conservative about "redundant" arithmetic even when it expresses unit semantics. Prefer `1 << 20` or a const named after the unit in future.

## Workarounds & Solutions

- **Contract typo (exit 35 vs 51)** — fixed in `exit-codes-p3.md` with a note that the variant name is canonical and the closed-enum mapping in `src/error.rs` is the authority. The corresponding test (`mcp_preflight_index_missing_exits_51`) asserts on the actual emitted code.
- **PR-D didn't address M-MCP-3 stderr fatal-line shape**. The contract specifies "one line, category + exit code"; the current `main.rs` path uses `output::write_error` which formats the human-readable error. The fix requires either a per-command stderr formatter or routing MCP-context errors through a different emit path. Tracked in PR-H disposition; not done in PR-D because the existing human stderr is at least readable and the cost-benefit didn't justify the rework mid-polish.
- **PR-D didn't address M-MCP-11 (subscriber install once-per-process)**. Documented as "intentional — `mcp::run` is one-shot per process by design". A `OnceLock`-style install gate would handle it but adds complexity for a non-load-bearing case.
- **PR-G deferred M-MCP-6 (unknown_plugin / unknown_skill MCP envelope tests)**, M-DOC-3 (schema-too-new through doctor), and the bulk of `m-WKS-*` test gaps. The marginal coverage gain didn't justify the test-setup cost given remaining polish-phase budget. Tracked in this retro for future phases.

## Patterns & Code

- **Four-reviewer pattern**. Dispatch contract audit / Rust-lens code review / test audit / security audit in a single message, each writing structured findings. Consolidate into one `findings.md`; triage into `disposition.md` with explicit per-finding actions. Generalisable to any pre-release pass; the per-reviewer focus areas in the disposition (blockers → PR-per-issue, majors → grouped PRs) scale.
- **Reviewer prompts include the "what NOT to do"**. Each reviewer was told "do not fix anything — just report". Without that, agents tend to write code; with it, they produce structured reports the orchestrator can synthesise.
- **Per-blocker dedicated PR**. Cheap to review, cheap to roll back. Phase 2 P10 also used this shape (PRs #34, #35 each closing one blocker class). Holds for Phase 3.
- **`#[doc(hidden)] pub static` for test injection**. F7's `MIGRATIONS_OVERRIDE` shape is now used in three integration test files (`schema_migration_e2e.rs`, `atomicity.rs`, `doctor.rs`). Documented as the test-injection idiom for compile-time-static framework lists in CLAUDE.md.
- **Custom `FormatEvent` for contract-pinned JSON shape**. `tracing-subscriber`'s default JSON output is sufficient for most cases; when contracts pin specific field names, a custom `FormatEvent` impl that builds the record via `serde_json::Map` is the right escape hatch. Reusable for any structured-output contract.
- **Closure-level `Err` as the SIGINT-mid-tx model**. Per-step transaction rollback is identical whether the abort came from a signal handler or a closure-level failure. Codified as a P5/P7 convention; PR-D's M-MCP-2 timeout testing benefits from the same idea (returning early from the closure mimics what a hard timeout would do).
- **`Orphan` informational variant + explicit classification rules**. The `match catalogs.iter().any(matches!(c.state, Missing | NotARepo | ManifestInvalid))` shape forces future variant additions to think about classification. Generalisable to any classify-and-suggest framework.
- **5-second graceful shutdown is "wait for the same future a second time"**. Pin the `RunningService::waiting()` future once; poll from both `select!` arms; after the signal arm fires, `tokio::time::timeout(deadline, &mut waiter).await`. Compiler-enforced once you pin.
- **Synthetic `SuggestedFix` injection for dispatch-path testing**. When a framework ships the dispatch + execute halves but doesn't yet produce the trigger, tests can inject the synthetic `SuggestedFix` to exercise the production code paths. PR-E's `repair_schema` wire-up is the production version of US5's test-only pattern.
- **MCP scrubbing at the log boundary, not the McpError payload**. Workspace paths and error chains go through `scrub_to_string` before logging. The McpError protocol payload is NOT scrubbed (the channel is already authenticated to the harness). Generalisable: scrub at "leaving Tome's trust boundary"; don't double-scrub.
