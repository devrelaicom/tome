## What Worked Well

- **Helper-visibility promotion to share state across diagnostic surfaces**: doctor's models / index / drift sections must agree with status's. The only safe way is to reuse the same compute. `commands::status::check_model` / `check_index` / `check_drift` got promoted from private to `pub` ŌĆö three one-word diffs ŌĆö and doctor's `assemble_report` calls them directly. Zero compute duplication. The two surfaces stay in sync by construction.
- **`home: &Path` as a test-isolation hook**: `doctor::assemble_report(scope, paths, home, verify)` takes the home directory as a parameter instead of reading `$HOME` inside. CLI wrapper resolves env, library is testable against a tempdir. No env mutation needed in tests; no `serial_test` requirement. Same idiom is reusable for any future read-from-env operation that wants test isolation.
- **`subsystem` string routing in `fixes::apply_one`**: each `SuggestedFix.subsystem` is its own dispatch key ŌĆö `"embedder"` / `"reranker"` / `"catalog:<name>"` / `"schema"`. The `.strip_prefix("catalog:")` extraction is one line; the dispatch chain in `apply_one` is a flat if-else ladder. No enum, no trait, no match-arm explosion. The repair classes are small and unlikely to grow.
- **`re_assemble` after `--fix` rather than another full `assemble_report`**: each `apply_one` re-ran the affected check function inline, so the per-subsystem report fields are already fresh. Only `suggested_fixes` + `overall` need recomputing ŌĆö `re_assemble` skips the catalog enumeration + harness probing that would have re-run unchanged. Cheap fix, real saving on the catalog side.
- **Surgical FS mutation for catalog-cache tests**: register a real `Fixture` via the CLI binary, then `remove_dir_all(cache.join(".git"))` to simulate NotARepo, or `write("garbage")` to the manifest for ManifestInvalid. Tests stay fast (no `git init` per case after the initial Fixture), deterministic, and exercise the real production code paths. No test-only mocking layer for the cache state.
- **`fabricate_all_installed_models` is the doctor-test baseline**: `cheap_state` checks both manifest presence AND file presence + size. The manifest-only `fabricate_models` helper produces `corrupt` state, forcing every "healthy baseline" doctor test to use the heavier sparse-file fabricator. Documented which helper suits which scenario in TESTING.md.

## What Didn't Work

- **First-pass tests used `fabricate_models` (manifest only)** and 5 tests immediately failed because `cheap_state` reported `corrupt`. The two fabricator helpers look similar by name but produce different on-disk states. Renamed every `fabricate_models` call to `fabricate_all_installed_models` in one sweep. Future test authors will hit this ŌĆö the TESTING.md guidance needs to be discoverable.
- **The `tome::doctor::report` types initially had `Eq + PartialEq` for round-trip testing**, but `IndexHealth` already had them so the derive chain composed without effort. Almost forgot that PartialEq on `f32` works only because no float fields exist in the doctor types ŌĆö keep this in mind if a future field adds a float.
- **`format!("{}", colour::success("Ō£ō"))` tripped clippy's `useless_format` lint** ŌĆö the `Style.paint(...)` value already implements `Display`. Changed to `.to_string()`. Three lines; flag for future colour usage.
- **The `--fix` CLI flow's exit code 75 path interaction with `std::process::exit(1)`** was subtle. Initial draft had `exit(1)` before the 75 check, masking the 75 path. Restructure: branch on `overall_ok` first (early-return Ok), then check `--fix + remaining_manual` (return DoctorFixNotSafe), then the `exit(1)` is reached only by the default-degraded/unhealthy path. The ordering is load-bearing.

## Workarounds & Solutions

- **Two `pub(crate)` shim functions for `classify` + `build_suggested_fixes`**: `re_assemble` lives in `doctor::fixes` and needs to call doctor's classify + suggested-fix builders. Making those `pub(crate)` lets fixes reach them without circular module visibility tricks. Documented as "not part of the public API."
- **`doctor::CatalogCacheState::as_str` not yet used in production code** (the human emit uses match-arms directly). Kept the method for future JSON-only consumers + tests. Small dead-weight; acceptable.
- **`schema` repair runs even when no migrations are registered**: Phase 2 ships with the migration framework but zero registered migrations. `apply_pending` returns 0 in that case ŌĆö no-op, no error. The `repair_schema` helper is forward-compatible with the first registered migration in Phase 4+.

## Patterns & Code

- **Helper-visibility promotion to keep diagnostic surfaces in sync**: when two surfaces (status, doctor) must report the same value for the same subsystem, promote the helper to `pub` rather than duplicating compute. The risk (helpers reachable outside the orchestrator) is mitigated by documenting them as "read-only diagnostic helpers." Generalisable to any future trio of surfaces that read overlapping state.
- **`home: &Path` as test-isolation hook**: when a library function would otherwise read `$HOME` directly, accept it as a parameter and have the CLI wrapper resolve env. Tests pass a `TempDir`-rooted path; production passes the real `$HOME`. Same idiom is reusable for any read-from-env operation tests need to isolate without `serial_test` (the `serial_test` crate is the alternative; this idiom is one step lighter).
- **`subsystem` string routing for tagged repair lists**: when a list of items needs per-item dispatch, encode the dispatch key in a string field rather than an enum. Prefix-strip (`.strip_prefix("catalog:")`) extracts per-instance data. Keeps the data wire-shape simple (a string) and dispatch logic flat (an if-else ladder). Trade-off: less type safety than an enum; pay only if dispatch arms grow > ~6.
- **`re_assemble` post-mutation pattern**: when a function mutates per-field state in place (`fixes::apply_one` updates `report.embedder` / `report.catalogs` / etc.), expose a second function that rebuilds derived state (`suggested_fixes`, `overall`) from those fields without re-running the expensive per-field compute. Avoids doubling the FS cost of a full re-assemble.
- **Existence-only directory probing**: `harness_detect::probe` reads only `is_dir()` for each fixed name in `KNOWN_HARNESSES`. No content reads, no config parsing, no `$HOME` scan. Privacy-sensitive surface kept maximally minimal. Reusable shape for any future presence-only probe.
