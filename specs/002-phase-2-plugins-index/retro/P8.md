# Phase 8 Retrospective: User Story 6 — Verify local installation health (status + version)

## What Worked Well

- **The library-bypass pattern for `std::process::exit` is clean.** `status::run` does `assemble_report` + emit + `exit(1)` for non-Ok. `pub fn assemble_report(paths, verify) -> StatusReport` is the pure function tests call directly. Same shape as `reindex::run_with_deps` from Phase 7. The pattern is now firm: when the CLI run() has a side effect that prevents test usage (process exit, ONNX load, etc.), expose a parallel library function. Document it once, reuse forever.
- **Pre-parse hook for `--version`.** clap's auto `--version` can't include compile-time constants beyond the package version, and can't honour `--json`. `disable_version_flag = true` on the Cli derive + a four-line `std::env::args` scan in `main.rs` BEFORE `Cli::parse()` solved both. The hook is small (~10 lines), self-contained, and follows the rustc/cargo idiom for `--version`. Pattern applies to any future flag that needs to short-circuit clap.
- **Sparse-file fixtures pulled their weight again.** The verify-checksum-mismatch test reuses `fabricate_all_installed_models` (from Phase 6); the all-zero contents disagree with the registry-pinned SHA, exactly what the test needs. Free fixture for a 280 MB reranker artefact.
- **Three-slice split (feat → tests → closeout) felt right.** Slice 1 was ~330 lines of production code + ~5 lines of helper promotion; slice 2 was ~370 lines of pure test code; closeout is docs. Each slice was inside the soft cap, each had a clear independent acceptance criterion. The split also let the bug surface — slice 2 caught two issues (`fabricate_models` not creating files, and the stub-vs-registry seed mismatch) that would have been harder to debug if interleaved with the production code.

## What Didn't Work

- **`fabricate_models` vs `fabricate_all_installed_models` confusion.** The lifecycle tests use `fabricate_models` (manifests only, no files) because lifecycle's model-presence check only reads manifests. Status's `cheap_state` walks `manifest.files` and verifies each file exists + has the right size. Initial slice-2 tests used `fabricate_models`, which made `cheap_state` classify both models as `Corrupt`. Switching to `fabricate_all_installed_models` (sparse files sized to `entry.size_bytes`) fixed it. **Lesson**: when borrowing a helper from a sibling test file, check what the helper does — not just what it's named.
- **Stub embedder seed vs registry seed mismatch.** Initial slice-2 drift tests used `stub_embedder_seed()` for `LifecycleDeps`, which bootstrapped meta with `stub-embedder` / `stub-reranker`. Status uses `registry_seeds()` (BGE names) for the configured side. Result: every test saw `EmbedderNameDrift { stored: "stub-embedder", configured: "bge-small-en-v1.5" }` regardless of the actual mutation. Fixed by passing `registry_seeds()` to the test's `LifecycleDeps` — the seed identifies the model in meta; the actual `StubEmbedder` produces vectors. The two are decoupled, but every prior test conflated them. Worth keeping in mind for any future drift test.
- **macOS rustup-shim flake hit again.** Phase 6 closeout (one runner), Phase 7 slice 2 (one runner), Phase 8 slice 2 (BOTH macOS runners simultaneously). The pattern is consistent: `cargo fmt --check` step finds `cargo` resolving to `rustup-init` from a cached shim, error message is `error: error: unexpected argument 'fmt' found`. `gh run rerun --failed` clears it. If this hits twice more, file an upstream issue.

## Workarounds & Solutions

- **Pre-parse hook for `--version`** (`src/main.rs`):
  ```rust
  let raw: Vec<String> = std::env::args().collect();
  if raw.iter().skip(1).any(|a| a == "--version" || a == "-V") {
      let json = raw.iter().any(|a| a == "--json");
      commands::status::print_version(json);
      std::process::exit(0);
  }
  let cli = Cli::parse();
  ```
  Combined with `#[command(disable_version_flag = true)]` on the Cli derive. Use this idiom anytime you need to short-circuit clap for a flag that needs to emit compile-time constants beyond the package version, or honour a global flag clap can't see at the auto-handler level.
- **Drift tests bootstrap with registry seeds, then mutate meta.** The trick is recognising that `LifecycleDeps.embedder_seed` identifies the model for the meta row but doesn't have to match the actual embedder. Pass `registry_seeds()` to the deps so meta records BGE names, then `write_meta(MetaKey::EmbedderName, "bge-OLD")` to simulate drift.

## Packages & Dependencies

- **No new dependencies.** Phase 8 reused everything from Phases 2–7: `serde_json` for the JSON path, `sha2` (re-exported via `embedding::download::sha256_file`) for `--verify`, `index::integrity::check` for the SQLite `PRAGMA integrity_check`, `index::meta::detect_drift` for drift classification.

## Patterns & Code

- **Library-bypass for `std::process::exit`.** Whenever a CLI run() has a process-terminating side effect, factor out the pure logic as a sibling `pub fn`. Tests use the sibling; CLI uses the run. Now firm in 3 commands (enable, reindex, status).
- **`--version` pre-parse hook** for compile-time-content version output. Patterned in `main.rs`. Cheap to add, easy to read, idiomatic for Rust CLIs.
- **Status classification policy.** Embedder failure or index integrity failure → Unhealthy (vectors invalid). Reranker failure or reranker drift → Degraded (queries still serve, just lower quality). Otherwise Ok. The asymmetry reflects the privacy/reproducibility model: embedder drift invalidates stored vectors; reranker drift only degrades ranking quality.
- **Compile-time-content tests.** `tests/version_output.rs` reads `MODEL_REGISTRY` at compile time and asserts the output contains those exact strings. A model bump auto-bumps the test. Use this pattern for any output that's parameterised by `&[Foo]` consts at compile time.
- **Helper promotion as a process.** Phase 8 promoted six items: `ModelState`, `cheap_state`, `read_manifest`, `primary_file_path`, `human_mb`, `registry_seeds` — all from `pub(crate)` to `pub`. The promotion is intentional: the type system tells us "this internal helper is now an external API surface". Worth a code-review checklist item.

## For Next Time

- **Document helper visibility intent at declaration.** When writing `pub(crate) fn foo`, drop a one-line `// candidate for `pub` if a test or sibling command needs this` so the next time we promote it's an informed decision, not a reflex.
- **Flag the "fabricate_models vs fabricate_all_installed_models" trap explicitly in tests/common/mod.rs.** Both helpers are alive; their semantics differ. A doc comment at the top of `fabricate_models` saying "manifest-only — use `fabricate_all_installed_models` for tests that need on-disk file existence checks" would prevent the next person from re-stumbling.
- **The macOS rustup-shim flake is recurring.** If it appears one more time, open an upstream issue against `dtolnay/rust-toolchain` with the three runs as evidence.

## Up-front context (slice 1 start)

### Goal

Phase 8 ships `tome status` (per-subsystem health report per `contracts/status.md`) and an extended `tome --version` (Tome version + embedder identity + reranker identity per `contracts/version-output.md`). Three slices:

- **Slice 1 (this slice)**: `src/commands/status.rs` (assemble + emit + exit-code semantics), `src/cli.rs` (`Command::Status`), and the extended `--version` handler (pre-parse hook in `src/main.rs` so `tome --version [--json]` short-circuits before clap dispatch).
- **Slice 2**: `tests/status.rs` + `tests/version_output.rs`. Library-API tests for `assemble_report` covering healthy / degraded (reranker drift) / unhealthy (model missing + integrity check failure). CLI binary tests for the `--version` plain-text + JSON forms.
- **Slice 3**: closeout.

### State going in

- `index::integrity::check(conn)` is the read-only integrity check.
- `index::meta::detect_drift(conn, embedder, reranker)` returns `DriftStatus::{None, EmbedderNameDrift, EmbedderVersionDrift, RerankerDrift}`.
- `commands::models::{ModelState, cheap_state, primary_file_path}` are already `pub(crate)` — slice 1 promotes them to `pub` so `status.rs` (a sibling under `commands/`) can reuse the cheap-state classifier instead of duplicating the manifest-versus-disk diff.
- `embedding::download::sha256_file(path)` already exists for the `--verify` rehash.
- `commands::plugin::{embedder_entry, reranker_entry, registry_seeds}` are `pub(crate)` and reachable from any sibling `commands/*` module.
- Phase 1 shipped clap's auto `--version` (single line, crate version only). Slice 1 replaces this with a pre-parse hook in `main.rs` so we can emit either the three-line plain-text form (no clap), or the JSON form when `--json` is also passed. `disable_version_flag = true` on the `Cli` derive prevents clap from intercepting `--version` first.

### What slice 1 does NOT ship

- Tests (slice 2).
- No new error variants. The status command communicates failure via its OWN exit-code surface (0 = healthy, 1 = degraded or unhealthy) — it does NOT propagate the per-subsystem exit codes (30/31/35/52 etc.) because that would prevent the report from rendering in the first place.
- No new dependencies.

### Soft size estimate

- `src/commands/status.rs`: ~280 lines (StatusReport struct mirroring data-model §11, assemble pipeline, human + JSON emit, exit semantics).
- `src/cli.rs`: +20 lines (`Command::Status` variant + StatusArgs struct + `disable_version_flag = true`).
- `src/main.rs`: +30 lines (pre-parse version handler).
- `src/commands/models/mod.rs`: ~5 lines (promote `cheap_state` + `primary_file_path` + `ModelState` to `pub`).
- Total slice 1 ≈ 335 lines. Inside the soft cap.
