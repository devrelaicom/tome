## MAJOR-1 (Rust-lens) — Agents sink symlink-refusal exit code diverges between its two removal paths (CON-1 regression)
**Where:** `src/harness/reconcile/agents.rs` — `emit_agents_for_harness` live-removal loop (~L318) vs `cleanup_all_owned_agents` orphan-cleanup (~L371-405).
**What:** R2/FR-007 gave the agents sink a dedicated exit-**45** (`AgentTranslationFailed`) symlink mapping. The write path (`write_agent_file`) and the live-removal path both run `refuse_symlinked_component` → 45. But `cleanup_all_owned_agents` calls `rules_file::remove_standalone` directly (no pre-check) → its refusal flows `refuse_symlink` → `map_err(TomeError::Io)` → **exit 7**. Same logical op (refuse to delete through a symlinked Tome-owned agent file) yields 45 when the harness is live, 7 when non-live/non-supporting. Violates exit-codes-p7.md:32/:37 + data-model.md:70 (dedicated sink → its code, never generic Io 7) — the exact Phase-6 CON-1 precedent recurring **within** the agents sink. **Phase-7 regression** (at b3bb263 both removal paths were symmetric at 7; R2 lifted the live path to 45, missed the cleanup parallel). Fail-closed either way (both REFUSE the unlink) — only the contract/exit-code differs. Untested (`tests/symlink_intermediate_guard.rs` has no agents case; the only 45 assertions are the in-crate `write_agent_file` unit tests).

## MAJOR-2 (Security) — SECURITY.md overstates model verification (aux files unverified)
**Where:** `SECURITY.md` ~l.24 vs `src/embedding/download.rs` (the aux-file stream loop) + `registry.rs` `aux_urls`.
**What:** Cross-slice drift: F-MODEL-FILES (P4) added the unverified aux-download path (tokenizer.json/config.json/special_tokens_map.json/tokenizer_config.json) to fix the headline beta-blocker; SECURITY.md's trust framing was authored later (REL4/P8) and never reconciled. Only the **primary** artefact (`model.onnx` + the summariser GGUF) is SHA-256-verified; the tokenizer/config sidecars fastembed actually consumes come down with **no hash pin**. The doc trades on a mechanical-guarantees framing for beta users, so a blanket "SHA-256-verified models" claim that doesn't hold for content Tome installs+loads is a real accuracy gap. Exploitability LOW (aux_urls are compile-time-pinned to HF) but the doc claim must match reality. Related minor: the aux stream loop has **no byte cap** (reads to EOF; aux files have no pinned size) — a compromised/MITM pinned host could serve an oversized sidecar (unbounded read).
