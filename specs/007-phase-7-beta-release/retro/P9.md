### MAJOR-1 (Rust-lens) — FIXED, merged #164
**Agents sink symlink-refusal exit code diverged between its two removal paths (a CON-1 regression).** R2/FR-007 lifted the agents sink's *live*-removal symlink refusal to its dedicated exit **45** (`AgentTranslationFailed`) but missed the parallel `cleanup_all_owned_agents` orphan-cleanup path, which still flowed `refuse_symlink` → `Io` (exit **7**). Same logical op (refuse to delete through a symlinked Tome-owned agent file) yielded 45 when the harness is live, 7 when non-live — violating the dedicated-sink → its-code rule (exit-codes-p7.md:32, the Phase-6 CON-1 precedent recurring *within* the agents sink). Fail-closed either way (both REFUSE); only the contract differed. **Fix:** run `refuse_symlinked_component` → map to `AgentTranslationFailed` (45) before `rules_file::remove_standalone` in the cleanup path, mirroring the live path; added an integration case in `tests/symlink_intermediate_guard.rs` driving the cleanup path through a symlinked agent file and asserting 45 (closes the contract + test minors too). Reused closed-set variant; no new code/schema/dep.

### MAJOR-2 (Security) — FIXED, PR #165 (open at this commit)
**`SECURITY.md` overstated model verification + the aux-download loop had no byte cap.** Cross-slice drift: F-MODEL-FILES (P4) added the unverified aux-file download path (`tokenizer.json` / `config.json` / `special_tokens_map.json` / `tokenizer_config.json`) to fix the headline blocker; `SECURITY.md`'s trust framing was authored later (REL4/P8) and never reconciled — only the *primary* artefact (`.onnx` + the summariser GGUF) is SHA-256-verified, but the doc implied a blanket "SHA-256-verified models" claim. The aux files (which fastembed actually consumes) come down with no hash pin and, before the fix, no size cap. **Fix:** (1) corrected `SECURITY.md` to state precisely that the primary artefact is SHA-256-pinned-and-verified while the auxiliary files are fetched from compile-time-pinned URLs but **not** hash-verified (keeping the mechanical-vs-semantic framing + the "catalog content is never vetted" line); (2) added a 64-MiB byte cap to the aux stream loop (`download.rs::stream_url_to_partial`), reusing the bounded-read discipline and mapping over-cap onto an existing error. Keeps the no-new-variant invariant. Exploitability was LOW (aux URLs compile-time-pinned to HF) — this is a doc-accuracy + defence-in-depth fix.
