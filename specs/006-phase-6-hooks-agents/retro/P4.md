# Phase 6 / Phase 4 (US2 — Real hooks for Claude Code) Retro

**Branch**: `006-us2-hooks`
**Baseline**: US1 merged @ fa8b25f (PR #132).

## What this phase landed

A plugin shipping `hooks/hooks.json` merges into `.claude/settings.local.json`
(never the committed `settings.json`) for Claude Code, with a targeted
two-variable plugin-root rewrite, structural-match (deep-equal) idempotent
add, structural-match removal that never touches user-edited entries, empty
event-array pruning, and atomic mode-preserving symlink-refusing writes.
Hooks reconcile FIRST in the sync order (hooks→guardrails→agents).

## What Worked Well

- **A single coherent subagent for US2** (hooks.rs + claude-code wiring + sync
  reconciliation + tests) was the right grain — smaller and more coupled than
  US1, so chunking would have added overhead. Gates green first pass.
- **The reconcile pattern generalised cleanly**: `reconcile_hooks` mirrors
  `reconcile_agents` (actions map + `first_error` forward-progress, error
  surfaced after `first_clash`), and the US1 R-1 lesson was applied
  proactively (`open_read_only(...)?` propagates, no silent `.ok()`).
- **Security review came back clean (0/0/0)** — the atomic symlink-refusing
  write reused from `mcp_config` and the structural-deep-equal ownership
  model held up; the reviewers' value this US was in test-coverage gaps.

## What Didn't Work

- The full 4-reviewer pass surfaced mostly TEST gaps (symlink refusal,
  forward-progress, exit-44 behavioral) for behaviour that was implemented
  but unasserted — the recurring lesson that happy-path subagent tests skip
  the failure-mode coverage.

## Workarounds & Solutions

- **Stale-hook-removal gap (no clean fix):** ownership is structural
  re-derivation with no sidecar (NFR-003), so a plugin whose source is
  evicted cannot have its previously-written `settings.local.json` entries
  re-derived for removal — they persist. Documented inline (both arms) +
  CONCERNS TD-063; US5 doctor `HooksReport` is the surfacing path.

## Packages & Dependencies

- No new top-level dependencies. JSON via `serde_json` (`preserve_order`);
  the two-token rewrite uses fixed-needle `str::replace` (the braces prevent
  prefix collisions) rather than `regex` — outcome-identical, ReDoS-free,
  accepted by the contract reviewer.

## Patterns & Code

- **Fail closed on non-UTF-8 load-bearing paths**: a path that becomes an
  *executed command* (the hook rewrite target) must refuse (exit 44) rather
  than `to_string_lossy` a U+FFFD-corrupted string. Distinguish load-bearing
  paths from display strings.
- **Reconcile-sink template** (`reconcile_<sink>`): read-only DB open
  (propagate on existing DB), compute shared inputs once, per-harness loop,
  actions map + `first_error`, append a `SyncSubsystem` variant +
  `<sink>_action` field LAST (byte-stable JSON pin preserved).
