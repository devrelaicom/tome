# Phase 6 / Phase 3 (US1 — Native agents across four harnesses) Retro

**Branch**: `006-us1-native-agents`
**Baseline**: Foundational merged @ fa9e5eb (PR #131).

## What this phase landed

Enabling a plugin with `agents/*.md` indexes agent-kind rows (non-searchable,
non-embedded) and emits native agent files into each natively-supported
harness's agent dir (claude-code/codex/cursor/opencode), plugin-namespaced
(`<plugin>__<name>.<ext>`), with same-vendor-only model mapping, read-only
intent reconstruction, dropped unsupported fields, clash-aware displayed
names, and glob-based removal. Reconciled on `tome harness sync` and
`tome plugin disable`; idempotent re-sync.

## What Worked Well
<!-- Effective patterns, tools, approaches -->

- **Sequential chunked subagents** (A indexing → B translation core → C
  per-harness+sync → D tests) kept each `devs:rust-dev` pass coherent and
  avoided same-crate compile races (parallel subagents editing one crate see
  each other's half-written code at the shared `target/` build lock).
- **Targeted-test verification per chunk** kept the loop fast; `cargo clippy
  --all-targets` compile-checks every target (incl. the `translate_agent`
  signature change propagating to StubHarness + integration tests) so a
  signature refactor is validated without the 25-min full suite.
- **The 4-reviewer pass earned its cost.** It caught a real **path-traversal
  BLOCKER** (attacker frontmatter `name: ../../evil` → write outside the
  agent dir) and a **data-loss MAJOR** (an existing-but-unopenable DB
  collapsing to an empty enabled set and wiping every emitted `<plugin>__*`
  file) — neither of which the happy-path targeted tests exercised.

## What Didn't Work
<!-- Problems, issues, limitations encountered -->

- The `typos` pre-commit hook repeatedly blocked commits on subagent-written
  comments that uppercase a SQL verb and append a lowercase suffix (an
  all-caps verb followed by a trailing lowercase `d`). Recurring; brief
  subagents to avoid it.

## Workarounds & Solutions
<!-- Bug fixes, non-obvious solutions, clever workarounds -->

- **Per-agent removal within an enabled plugin** (shrinking agent set) is a
  known FR-043 narrowness: removal is keyed on plugin-enablement, so a stale
  `<plugin>__<oldname>` survives until full disable. Defensible under the
  literal contract; US5 `tome doctor --fix` removes orphaned agent files.

## Packages & Dependencies
<!-- Useful packages discovered, packages to avoid, version issues -->

- No new top-level dependencies. Codex TOML via existing `toml_edit`
  (content-driven triple-quoting: a body with newlines renders as a `"""…"""`
  multiline string; single-line bodies render as a basic string).

## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->

- **Validate third-party names as a single safe path segment at the
  boundary** before composing them into a path: `is_safe_agent_name` rejects
  empty/NUL/`/`/`\`/`.`/`..`/leading-dot and requires exactly one
  `Component::Normal` equal to the name. Pair with a defensive
  `target.parent() == Some(dir)` check at the write site. Reusable wherever
  plugin-supplied strings become filenames.
- **Same-vendor-only model alias table** (`map_model`) with drop-on-no-target
  — never cross-vendor (SC-002). The policy slot exists per harness even when
  the id set is not yet enumerated (Cursor returns None for now).
- **`plugin_of_owned_file` SSOT** for the `<plugin>__*` provenance/removal
  rule — one parser, consumed by both emission and cleanup.

## For Next Time
<!-- Improvement suggestions, lessons learned, things to try differently -->

- Tell subagents up front: avoid `typos`-tripping comment prose; leave no
  background processes; verify with targeted tests not the full suite.
- The security lens is worth dispatching every US — a happy-path test suite
  will not find a path-traversal or a destructive-cleanup-on-error bug.
