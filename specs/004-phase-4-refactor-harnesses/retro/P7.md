## What Worked Well

- **Smallest reviewer-pass surface** of any US so far: 1 blocker + 21 majors vs US3's 6 blockers + 29 majors and US4's 5 blockers + 21 majors. The doctor surface was conceptually well-defined (data-model.md §15 specified the wire shape exhaustively); reviewers caught real edge cases without flagging architectural issues.
- **Typed `Subsystem` enum with byte-stable wire** — the P6 retro recommended promotion at >6 distinct values; US5.a delivered 11 variants with custom Serialize/Deserialize that preserves the Phase 3 string shape byte-for-byte. Round-trip test (T375) locks every variant. Generalisable pattern: any closed dispatch ladder above the 6-variant threshold promotes to enum with custom serde that preserves wire.
- **4-mapper parallel doc refresh** continues to scale: ~5 min wall-clock for all 8 codebase docs across 4 concurrent agents. Used at every US closeout (US1, US2, US3, US4, US5) — predictable cost.
- **Reviewer pass per US** caught the orphan-cleanup `remove_dir_all` attack surface BEFORE it shipped to main. Security reviewer confirmed STAGING_PREFIX + 1h mtime + symlink-skip + is_dir() + 0o700 staging perms compose correctly. Without the reviewer pass, the defence-in-depth combination could have shipped with one layer accidentally weakened.
- **`SubsystemHealth::NotApplicable` patched same session** (US5.c-1 C-M1): contract auditor flagged the variant as declared-but-never-used; the fix took 4 lines + 2 test updates. Pattern: when a data-model document specifies an enum variant with explicit semantics, the implementation MUST actually emit it; reviewer pass catches mis-implementation.
- **Coalesced repair invocations** (US5.c-1 R-M2): the original implementation invoked `sync_for_project_root` per HarnessRules + per HarnessMcp suggestion (10× redundant for 5 harnesses). Fix collects matching suggestions, runs the repair once, clears all affected from residual. Generalisable: any `apply` loop that dispatches multiple suggestions should coalesce per-target before invoking the underlying repair.

## What Didn't Work

- **`SubsystemHealth::NotApplicable` declared but unused** (C-M1): same pattern as US4's placeholder SHA — the data-model documented the variant, the implementation acknowledged it in doc comments, but the actual emission point shipped with empty Vec. Reviewer caught it; we shouldn't rely on the reviewer pass for "documented-but-unused" gaps. **Lesson**: when a slice ships an enum variant with documented semantics, the SAME PR must include at least one test exercising the emission path.
- **`SchemaTooNew(52)` propagated from `assemble_report`** (C-M2): every other check function in `assemble_report` collapsed gracefully; only `check_index` short-circuited with `?` on error. Reviewer flagged the FR-561 "doctor never crashes" violation. **Lesson**: when a contract says "X never crashes", the implementing slice must explicitly audit every `?` in the X codepath; reviewer pass shouldn't be relied on for this kind of invariant audit.
- **`repair_binding_rules_copy` used workspace-broadcast `sync_one`** (C-M3): the project-local repair was meant to copy ONE project's RULES.md but reached for the workspace-broadcast helper that walked every bound project. Reviewer caught it; fix added a new `sync_one_project` helper. **Lesson**: when a helper has both single-target and all-targets variants, the API should make the choice explicit at the call site (not via a parameter). The single-target case is often easier to overlook.
- **`RulesCopyState::Missing` collapsed two distinct failures** (R-M5): "workspace source absent" vs "project copy absent". `--fix` would infinite-loop attempting to copy from an absent source. Fix added `SourceMissing` variant + non-auto-fixable SuggestedFix. **Lesson**: when a state enum has multiple distinct failure modes that need different recovery, distinguish them at the type level. Don't collapse for "simplicity" then trip on the divergent recovery.

## Workarounds & Solutions

**Subsystem enum with byte-stable wire** (US5.a T364): 11 variants; custom Serialize emits "embedder" / "catalog:<name>" / "harness-rules:<name>" etc.; custom Deserialize splits on `:` and routes to the appropriate variant. Round-trip test (T375) asserts every variant survives serialize → deserialize → eq. Generalisable shape for any future `Subsystem`-style closed dispatch.

**`SubsystemHealth::NotApplicable` for inapplicable per-row checks** (US5.c-1 C-M1): when the doctor reports per-harness state but no project is bound, the harness rows still emit (with `NotApplicable`) rather than empty Vec. Wire distinguishes "no harnesses declared" (empty Vec) from "harnesses declared but not in project context" (NotApplicable entries). Reusable for any per-item health that needs a skip-but-report state.

**Coalesce repair invocations** (US5.c-1 R-M2): in `fixes::apply`, collect all `HarnessRules(name)` + `HarnessMcp(name)` suggestions; invoke `sync_for_project_root` ONCE; clear all affected from residual. Pattern: when multiple suggested fixes share a repair function targeting the same resource, group by (target, function) before invoking.

**Project-local vs workspace-broadcast naming** (US5.c-1 C-M3): both `workspace::sync::sync_one(workspace_name, paths)` (broadcasts to every bound project) and `sync_one_project(project_root, source, paths)` (targets one project) coexist. Call site chooses by NAME, not by a parameter. Prevents future readers from mistaking which semantic applies.

**`SourceMissing` vs `Missing` distinct variants** (US5.c-1 R-M5): when a copy-from-source state has two failure modes, distinguish them in the enum. `--fix` dispatcher pattern-matches on the variant and applies the appropriate response (auto-fix the copy-missing case; surface manual SuggestedFix for source-missing).

**`--force` precise scoping** (US5.c-1 S-M2): `--force` rewrites only the conflicts in the CURRENT run's suggested-fix list. Filter the user-owned set by active fix entries before bulk-rewrite. Prevents blanket-rewrite scope creep.

**`debug_assert!` for safety invariants on DB-derived paths** (US5.c-1 S-M4): `debug_assert!(cache_dir.starts_with(&paths.catalogs_dir))` before `remove_dir_all`. Documents the invariant + catches future refactors that break it. Pattern: every `remove_dir_all` on a path derived from external data carries a starts_with assertion.

**Orphan cleanup defence-in-depth composition** (US5.b): STAGING_PREFIX literal match + 1h mtime gate + `DirEntry::metadata` (no symlink traversal) + `is_dir()` check (rejects symlinks-to-dirs) + 0o700 staging perms from `atomic_dir` (limits who can plant orphans). Each layer alone wouldn't be sufficient; the composition is. Pattern: any `remove_dir_all` against a scanned-directory result combines these five layers minimum.

## For Next Time

- **Audit `?` in every "<X> never crashes" code path**: when a contract specifies a non-crash invariant, the implementing slice's PR description should include a `grep -n '?' src/<path>/<file>.rs | wc -l` reality check.
- **Documented-but-unused enum variants caught by reviewer**: when a data-model spec says variant X has semantics Y, add a test that exercises the emission path in the SAME PR as the variant. Don't ship "this variant exists for future use" without a test.
- **`--force` flag introduction checklist**: any future `--force` flag MUST explicitly scope its destructive effect (which fixes are in scope), document the override path (where the user reads about it), and test BOTH the without-force refusal AND the with-force rewrite in the same PR. Reusable from US5's `tome doctor --fix --force`.
- **5-layer `remove_dir_all` discipline**: when a future feature does `remove_dir_all` on scanned-directory results, compose at minimum: name-pattern match, mtime gate, no-symlink-traversal stat, is_dir check, restrictive parent perms. US5.b's orphan cleanup is the reference pattern.
