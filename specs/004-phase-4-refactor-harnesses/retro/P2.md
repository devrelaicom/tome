## What Worked Well

- **Pre-allocated error variants (F3 pattern)**: 8 new variants landed in F3 with codes 13–19, 24 *before any consumer existed*. Each subsequent slice that needed one (F9, F11) just wired the existing variant — zero closed-enum churn under load. Same playbook as Phase 2 / Phase 3 worked again. Worth carrying to Phase 5.
- **Folded-in deferred items survived re-folding**: the plan deliberately picked up 7 Phase 3 deferred items. F2a took the read-only DB open refactor. F6 took the byte-progress download callback. F10's WorkspaceName scope reshape consumed the `Paths.config_file` field rename by deletion. The "fold deferrals into the next phase's foundational" pattern landed cleanly; only one Phase 3 deferral (manual SC-001/SC-002 against real models) survived into Phase 5+.
- **Splitting F11 into F11 / F11b / F11c-1 / F11c-2** after the F11b agent timeout. The original brief was too large for one PR; the agent ran 80 minutes / 105 tool uses and got 80% of the way before timing out. The recovery — commit the WIP as "F11b rewire only", queue F11c-1 (catalog tests) + F11c-2 (semantic + audit) — preserved every dropped task. Final F11 closed in 5 PRs instead of 1, but the work shipped.
- **Test helper promotion at the rule-of-3** continues to pay off. `seed_workspace`, `cache_dir_for`, `paths_for`, and `registry_seeds_for_test` all migrated to `tests/common/mod.rs` at the 3rd-or-4th caller — never earlier, never later. Reuse without speculative abstraction.
- **`#[ignore = "<unhide target>"]` markers** with explicit unhide targets (F11 / US1 / US4.b / etc.) survived three months. Every ignored test landed cleanly when the named slice arrived, or was deleted with documented reason. The discipline of always naming the unhide target prevents test-suite rot.

## What Didn't Work

- **F11 scope estimation was off by ~3×**. The handover described F11 as "small" (lift the global filter, widen refcount, unhide tests). The actual tasks.md listed 19 sub-tasks covering the full junction-table rewire. Reading tasks.md before estimating would have caught this. The agent timeout on F11b was a direct consequence — a more accurate scope would have predicted F11 as 4 PRs from the start, not 1.
- **Agent dispatch with very large briefs** — F11b's brief was ~12 KB. The agent worked through most of it but burned all its budget. For slices over ~8 KB of brief, split into sub-slices BEFORE dispatching, not after the timeout. This is the F9 retro's "interrupt at 20 min if agent goes silent" lesson generalised: also bound the work surface BEFORE handing off.
- **`config.toml` excision is deferred** — F11b left `Config.catalogs` in place as a transitional shell that's never written. The full struct excision is a follow-up cleanup PR. It compiles + tests pass, but it's dead code on disk. Future phases should treat "leave deprecated shells in place" with suspicion — they rot.

## Workarounds & Solutions

**F3 — variant name + exit-code collisions** (commit `0f10079`):

1. *Variant name collision*: Phase 3's `WorkspaceNotFound { path: PathBuf }` (exit 71, the marker-walk error) shared the `WorkspaceNotFound` name with the Phase 4 spec's new `WorkspaceNotFound { name: String }` (exit 13). Resolved by renaming the Phase 3 variant to `WorkspaceMarkerMissing` with a doc-comment flagging it as transitional. F10 deleted it.

2. *Exit-code collision*: early draft `contracts/exit-codes-p4.md` specced `SummariserFailure` at code 20, but Phase 2's `PluginNotFound` already owns 20. Closed-set rule II (pairwise-unique codes) forced a move. `SummariserFailure` placed at code 24 (next free slot above the Phase 2 plugin-lifecycle range 20–23). Contract + data-model reconciled in the same PR (commit `632a281`).

**F5 — audit findings** (no code change):

- *`serde_json/preserve_order`*: enabled project-wide in PR #59. Audit of `src/` shows no codepath relies on alphabetical `Value::Object` ordering: `output.rs` / `embedding/download.rs` / `plugin/lifecycle.rs` / `commands/models/mod.rs` / `commands/plugin/mod.rs` serialise typed `Serialize` structs (struct-field declaration order, independent of map backing); `plugin/components.rs` `from_slice::<Value>` only structurally detects fields without writeback; `mcp/log.rs` uses typed `serde_json::Map` directly. Switching from `BTreeMap` to `IndexMap` is therefore behaviourally neutral on existing code. F7's harness MCP config read-modify-write paths are the first consumers of the order-preservation guarantee.
- *`toml_edit`*: zero imports under `src/` at HEAD. F7's `harness/mcp_config.rs` is the first consumer. Tome-owned TOML (`config.toml`, `settings.toml`, manifests) continues to use the `toml` crate via `serde`.

**F10 — `plugin::lifecycle` test_scope used `OnceLock`** (PR #68): the brief suggested `fn test_scope() -> Scope` with local rebinding at call sites. Because `LifecycleDeps.scope` is `&'a Scope`, a function returning a value forces every caller to introduce a local binding. The agent used `static TEST_SCOPE: OnceLock<Scope>` with `fn test_scope() -> &'static Scope` instead — every existing call site stayed shape-compatible (`scope: test_scope()` replaces `scope: &TEST_SCOPE`), no per-site rebinding. Generalisable: when retrofitting a tuple-struct or newtype into a position currently held by a static enum value, `OnceLock` + accessor preserves the `&'static T` shape.

**F11b — agent timeout recovery** (PR #70): agent worked 80 min / 105 tool uses on the catalog rewire and timed out before creating new test files (T098o, T098s) and unhiding ignored tests. Recovery: inspected WIP (build clean, tests pass, just clippy + fmt cosmetics to fix), fixed the residuals on the main thread, committed as "rewire only", and queued the deferred test work as F11c-1. **The lesson generalised: a partial agent dispatch is usually salvageable if (a) the build is green and (b) tests pass.** Don't re-dispatch the same brief — debug the residual and ship the partial.

**F11c-2 — `MIGRATIONS_OVERRIDE` thread-local doesn't cross `thread::spawn` boundaries** (PR #72): the MVCC reader test spawned a writer thread to drive `apply_pending`. `MIGRATIONS_OVERRIDE` is `thread_local!`, so the writer's `index::open` fell back to the production registry — which now has `phase_4_v1_to_v2` registered and tried to `SELECT FROM skills` on the meta-only synthetic DB. Fix: install the guard per-thread (`MigrationsGuard::install(MIGRATIONS)` inside the spawned closure). Generalisable: any test that spawns threads through `MIGRATIONS_OVERRIDE` needs the guard per-thread.

**F11c-2 — WAL required for SQLite MVCC snapshot** (PR #72): the existing `write_index_db_with_schema_version` helper defaults to rollback-journal mode. The reader-during-writer test failed deterministically until WAL was enabled on the synthetic DB. Pattern: any test asserting reader-during-writer or snapshot-isolation behaviour MUST explicitly enable WAL.

**CI transients (PR #69 ubuntu MSRV + PR #72 cargo deny)**: both failed with infrastructure errors (linker bus error on the runner; checkout auth failure). Both passed on retry. The bus error in particular is correlated with MSRV builds (heavier than stable). Worth noting for future slices: **a single ubuntu-msrv failure with a `collect2: fatal error: ld terminated with signal 7` is almost always a runner-OOM, retry-don't-debug.**

## Patterns & Code

- **Pre-allocated error variants** (F3): name + slot variants in the closed `TomeError` enum BEFORE consumers exist. Each subsequent slice wires variants without churning the enum. Pattern carried from Phase 2 and Phase 3.
- **`workspace_name: &str` parameter convention** (F11a): every SQL helper that touches `workspace_skills` / `workspace_catalogs` takes the resolved workspace as a `&str` parameter at the helper boundary. Source: `scope.scope.name().as_str()` at command-layer entry. `WorkspaceName` newtype enforces validity; `&str` is the SQL-binding shape.
- **`LifecycleDeps::workspace_name(&self) -> &str` accessor** (F11a): introduced at the rule-of-3 (`enable_locked`, `reindex_locked`, `any_skill_enabled` all needed it). Encapsulates the `scope.name().as_str()` lookup so callers don't repeat it. Saved a 6-character noise expression across ~12 call sites.
- **`OnceLock<T>` static + accessor for `&'static T`** (F10): when retrofitting a value-typed newtype into a position currently held by a static enum value (`Scope::Global` as a literal in `static TEST_SCOPE`), `OnceLock<NewType>` + `fn accessor() -> &'static NewType` preserves the `&'static T` shape at every call site. Avoids per-site rebinding.
- **Workspace seeding test helper** (`seed_workspace(paths, name)` promoted to `tests/common/mod.rs` in F11c-1): inserts a row directly into the `workspaces` table. Signposts the seam US1/US2 owns when `tome workspace add` ships in production.
- **Sole-source-of-truth + derived-metadata** (F11b): when migrating an authoritative on-disk file (`config.toml`'s `[catalogs]`) to a DB table (`workspace_catalogs`), derive secondary metadata from the filesystem (mtime, manifest read) rather than duplicating into the new DB schema. Halved the schema change surface and avoided per-row staleness.
- **Synthetic SuggestedFix injection retired** (F9): the framework now emits `subsystem: "schema"` natively when the on-disk schema lags. Phase 3's `tests/doctor.rs::fix_runs_forward_schema_migration_end_to_end` manual injection is gone. Pattern: when a framework's production-side trigger logic ships, retire the synthetic-injection test fixtures it was a proxy for.
- **Atomic-populated-directory helper promotion** (F4): the `tempfile::Builder::tempdir_in(parent) → populate → keep() + std::fs::rename` pattern landed in `src/util/atomic_dir.rs` with both `land_directory` and `land_directory_with_replace`. The `_with_replace` variant adds rename-aside-then-rollback semantics for replace-existing flows.
- **Schema-migration framework with synthetic fixtures** (F7 framework + F9 first real migration): `MIGRATIONS_OVERRIDE` thread-local lets tests register synthetic migrations against the real `apply_pending` machinery. Generate-at-setup fixture DBs (`write_index_db_with_schema_version`) avoid committed binary `.db` blobs. F9's `phase_4_v1_to_v2` is the first production migration; rails were battle-tested before.

## For Next Time

- **Estimate slice size from tasks.md, not handover prose.** When a handover says "small slice", reach for tasks.md and count sub-task identifiers. F11's "small" → "19 sub-tasks" was the F11b agent timeout.
- **Cap agent brief size at ~8 KB**. F11b's 12 KB brief exhausted the agent budget before completion. For larger slices, pre-split into sub-PRs.
- **`workspace_name: &str` parameter threading is the norm post-F11a**. When Phase 5+ adds new SQL helpers, default to the `&str` parameter at the boundary; don't reintroduce hard-coded literals.
- **`OnceLock<T>` accessor pattern** is the answer when a value-typed type needs to substitute for a static enum literal. Carry forward into future newtype retrofits.
- **`MIGRATIONS_OVERRIDE` is thread-local** — any future test spawning writer threads needs per-thread `MigrationsGuard::install`. Documenting once.
- **WAL is the default for snapshot-isolation tests**. Add a comment to `write_index_db_with_schema_version` noting that callers asserting MVCC behaviour must enable WAL.
- **Sole-source-of-truth + derived-metadata** (F11b pattern): when the next phase moves another `config.toml` section into the DB, use the F11b template — junction table for enrolment, filesystem for secondary metadata.
- **Cargo deny / cargo audit checkout failures are transient.** Retry once before debugging. Same for ubuntu MSRV `collect2: ld terminated with signal 7 [Bus error]` — runner OOM, retry.
- **Codebase doc refresh via `/sdd:map incremental`** is cheap (~3 min for 4 parallel mapper agents producing 8 docs) and worth running at every phase closeout. Keeps `.sdd/codebase/` in sync with the merged source.
