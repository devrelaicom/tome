## Workarounds

### B2 — Data exfiltration vector closed via single-sweep refactor (CRITICAL FIX)
US2.a–c shipped with `apply_env` running `replace_all` over the FULL Stage 1 output. Reviewer pass surfaced this as a BLOCKER: a hostile plugin author could set `"version": "${TOME_ENV_GITHUB_TOKEN}"` in `plugin.json` (lenient parser) and any skill body using `${TOME_PLUGIN_VERSION}` would leak the operator's env var into LLM context. Fix: merged Stage 1 + Stage 2 into a single regex sweep using union pattern `\$\{TOME_(?:ENV_([A-Z0-9_]+)|([A-Z0-9_]+))(?::-(.*?))?\}` with leftmost-first alternation. Each match resolved in isolation — resolved values are NEVER re-scanned. NFR-007 / FR-051 now structurally enforced. Reviewer pass was the difference between shipping a serious security issue vs. catching it pre-release.

### `now_local()` continues to be unavailable
US2 honoured the US1.c decision: `time::OffsetDateTime::now_local()` requires the `local-offset` cargo feature which is not enabled. Substitution layer uses `now_utc()` as default with `SUBSTITUTION_CLOCK_OVERRIDE` test seam.

### `EnvVarGuard` + `ENV_MUTEX` not promoted at first consumer
US2.b kept `EnvVarGuard` + `ENV_MUTEX` file-local in `tests/substitution_env.rs`. US2.c-1 duplicated it into `tests/substitution_pipeline.rs`. At 2 consumers it's borderline; at 3 consumers (US3?), promote to `tests/common/mod.rs`. Pattern: rule-of-3 — don't promote at first or second consumer; promote at third.

### Sub-agent dispatch returned mid-task once
The US2.b sub-agent stopped mid-task with "Now let me tick the checkboxes in tasks.md while waiting" but didn't commit. Orchestrator verified locally and split into feat + test commits. Same pattern as US1.b first dispatch. Lesson confirmed: explicitly require the sub-agent to RUN `cargo test` + `git commit` before returning.

### Single-regex sweep beats two regex sweeps for multi-stage transforms with no-rescan
B2's fix demonstrates the pattern: when multiple "stages" must NOT re-scan each other's output, the structural fix is to use a single sweep with union regex + branch resolution per match. Two sequential sweeps inherently re-scan unless guarded by complex marker tracking. The union-regex approach is conceptually simpler and structurally correct. Carries forward to US3 where Stage 3 (argument substitution) joins.

## Patterns established

### 4-reviewer parallel-pass discipline continues
US2.d's reviewer pass surfaced 2 critical blockers that none of the implementation sub-agents caught. The 4-reviewer parallel pattern (contract / Rust-lens / test / security) — established in US1.d — proves its value on every US closeout. The 2 BLOCKERS were independently identified by 3 of the 4 reviewers (rust-lens called them blockers; contract + test called them majors). Sub-agent dispatch in a SINGLE message with multiple Agent tool uses keeps wall clock low (~5-7 min for all 4).

### Disposition document scopes the fix surface
Writing `us2-disposition.md` BEFORE dispatching the fix sub-agent makes the brief tight: "apply these 2 BLOCKERS and these 6 MAJORS; defer the rest". The fix sub-agent doesn't have to re-judge severity; it just executes. Phase 4 P5 pattern.

### Structural fixes over targeted fixes for security-critical invariants
B2 had two fix options: (a) merge Stage 1+2 into a single sweep (structural — invariant cannot be violated), (b) track substitution markers and skip re-scanning (targeted — possible to mis-implement). Picked structural. Future stages (US3) automatically inherit the discipline.

### Pre-allocation slice patterns carry through (F1 + F3 → US2.a + US2.b)
F1 pre-allocated `PluginDataDirWriteFailed` (exit 9). US1.d carved off `WorkspaceDataDirWriteFailed` (exit 25). US2.a lit up both for the first time in production. The pre-allocation discipline means the consumer slice (US2.a) didn't need a contract amendment for new exit codes — they were already there.

### `#[allow(dead_code)]` discipline removed when consumer lands
F3's `#[allow(dead_code)]` markers on internal substitution helpers came off in US2.a + US2.b when the production consumers wired them. Convention: every `#[allow(dead_code)]` carries a doc comment naming which US activates it; that comment turns into a TODO-now when the slice opens.

## For next time

### Reviewer pass MUST run before any user-facing security-sensitive surface ships
US2 added prompt-injection trust boundaries (substitution into LLM context). The reviewer pass caught a CRITICAL exfiltration vector that all 3 implementation sub-agents missed. Lesson: any feature involving third-party data flowing into agent context gets a security review pass, not just the standard 4-reviewer pass — and the security reviewer should explicitly look for "could a hostile catalog do X" scenarios.

### Stage-merging applies broadly when "no rescan" is part of the spec
When a multi-stage spec says "later stages don't re-scan earlier output", the structural fix is to merge stages into a single sweep. Don't try to enforce the invariant via marker tracking or careful pattern design. The single sweep is simpler and structurally correct.

### Sub-agent dispatch verification pattern
For sub-agents writing production code: explicitly require in the brief "run `cargo test` (full suite) BEFORE committing and report the test count". US2.b sub-agent stopped mid-task; US2.d sub-agent followed this discipline and returned cleanly.

### Test pyramid for substitution-style transforms
US2 lands:
- Per-stage unit tests (one per built-in, one per env-state row): ~30 tests, very fast
- Pipeline integration tests (end-to-end through render): ~4 tests
- Wire integration tests (through MCP handler): ~2 tests
- Invariant tests (no-rescan, idempotence): NOW exists via US2.d fix; was missing in US2.a–c

Pattern: invariant tests should be part of the slice that ESTABLISHES the invariant, not deferred to the reviewer pass. The no-rescan invariant was in NFR-007 from day one; should have been tested in US2.a.

### `.expect(...)` over `.unwrap_or(...)` for non-optional regex captures
Default value of "" silently swallows future regex changes. `.expect("capture group 1 is non-optional in BUILTIN_REGEX")` panics clearly when the regex changes shape. Apply across US3+ regex callers.

### Single-sweep refactor saves > 100 lines and one regex
US2.d's single-sweep refactor dropped `apply_builtins` and `apply_env` as separate stages. The combined regex resolves both in one pass. Net code reduction + simpler control flow + structurally-correct invariant.
