# Phase 5 / Phase 3 (US1) Retro — Commands as MCP prompts

**Slices**: US1.a (schema migration + frontmatter widening), US1.b (MCP prompts capability), US1.c (prompts/get + substitution wiring), US1.d (reviewer pass + closeout)
**Phase 2 baseline**: PR #111 merged @ 0d3c391. Test total 961/128/16.

## What this phase will land

The headline Phase 5 deliverable: commands accessible as slash commands across
harnesses via the MCP prompts capability.

- Schema v2 → v3 migration (kind/searchable/user_invocable/when_to_use columns)
- Lenient frontmatter parsing for new fields (`arguments`, `when_to_use`,
  `searchable`, `user-invocable`)
- `EntryKind` enum (Skill | Command) at the type level
- Commands directory walk in `plugin::components`
- MCP `prompts` capability (`prompts/list` + `prompts/get`)
- Prompt name derivation (`<plugin>__<entry>` format with sanitisation +
  truncation + collision counter-suffix)
- Wiring substitution::render() into prompts/get (still a no-op stub from F3;
  US2 + US3 wire real transforms)
