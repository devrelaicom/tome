## Workarounds + surprises

- **The Polish reviewer pass cost ~5 min wall-clock for 4 parallel agents but produced 7 majors + 14 minors against the entire Phase 5 surface.** Each per-US pass cost the same wall-clock and produced 0-2 majors against a single slice. **The phase-wide pass is structurally a better catch ratio** because it sees the cross-US drift the per-slice reviewers can't. This isn't a cost-saving observation (per-US passes catch issues earlier when they're easier to fix); it's a value-add observation: the Polish pass should run on EVERY phase even when per-US passes were thorough.

- **R-style M-1 (truncate_description pattern drift) is the kind of finding that only the cross-US pass catches.** The per-US pass for US1 (which shipped `prompts.rs`) saw correct code at the time; the US4.d fix landed later in `search_skills.rs`. No single reviewer was looking at both functions in the same pass; the Polish reviewer caught the divergence by re-walking the full tree. **This is the canonical kind of cross-US drift the Polish phase is designed to catch.**

- **The disposition's `GAP-1` partial deferral** is a documented carve-out, not a regression: codes 9 + 26-29 + 25 are all production-reachable but the test scaffolding cost is high (MCP-server in-process driving for 9/26-29; workspace-rename-with-data-subtree-and-unwritable-target for 25). All six are exercised at the library API level via `tests/exit_codes.rs` + `tests/mcp_prompts*.rs`. The v0.6+ work is shipping a unified `tests/exit_codes_e2e_mcp.rs` once the test harness exists.

- **M-4's promotion of `validate_db_stored_path` had a subtle pitfall**: my first edit placed the new helper between `resolve_entry_body_path`'s docstring and its body. The docstring then attached to the helper instead of the function it described, and clippy's `doc_list_item_without_indentation` lint caught it via the bulleted list at the docstring's end. Easy fix (move the helper AFTER `resolve_entry_body_path`), but worth noting: **clippy is a docstring-attachment checker, not just a code checker**. The fix is permanent: helpers go AFTER the function whose docstring would otherwise capture them.

- **PR-D Cargo.lock auto-updated** when the Cargo.toml version changed. No manual edit needed; the lock file's `version` for `tome` follows the manifest.

## Patterns established (Polish-specific)

- **Phase-wide 4-reviewer pass at Polish start** mirrors the per-US closeout passes but with a cross-cutting brief. Each reviewer reads the per-US disposition docs FIRST (`us{1..5}-disposition.md`) so they don't re-litigate already-decided items. The brief explicitly names "this is the cross-cutting check" so reviewers look for drift, not new findings within slices.

- **Findings.md tally format** carries the per-US tally trend (US1.d through US5.c) so readers can see if Polish is finding more or fewer issues than the per-US passes. Trend-wise: Polish typically catches the cross-US drift that per-US can't see; the absolute count is higher than the latest US but lower than US1 (the start-of-phase pass when patterns weren't established).

- **Disposition.md "Applied — PR-X" sections** explicitly name which PR carries each fix. Makes the closeout commit messages easy to write + makes the audit trail navigable. Carry-forward backlog goes at the bottom; v0.6+ items live there.

- **Cross-US single-source-of-truth promotion at "second consumer".** The rule across Phase 5: when a layout/string/path/predicate appears at TWO sites with the same shape, promote to a canonical accessor. Phase 5 racked up 5 of these:
  - `Paths::plugin_data_root()` (US5.c R-M1)
  - `Paths::workspace_data_dir_for(workspace, catalog, plugin)` (US1.c)
  - `MCP_SLASH_PREFIX` (US5.c R-m11)
  - `validate_db_stored_path` (Polish M-4)
  - `build_context_for_entry` (Polish M-2)

  This is now the canonical pattern for any duplicated layout/predicate at v0.6+.

- **Bounded `char_indices` walk for caller-controlled string truncation** is the canonical shape. Polish PR-B M-1 propagated US4.d's HIGH-severity fix to the second truncate site. v0.6+ should grep for any new `chars().count() <= max` patterns and replace.

- **Canonical enum dispatch over stringly-typed match.** Polish PR-B M-3 fixed two `match kind.as_str()` sites. v0.6+ should grep for any `kind.as_str()` patterns in new code and reject in review.

- **CHANGELOG entry structure** for a phase release: User-visible → Internal additions → Bug fixes → Exit codes → Dependencies → Tests → Polish notable. Phase 5's [0.5.0] entry sets the precedent.
