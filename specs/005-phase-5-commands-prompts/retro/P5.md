# Phase 5 / Phase 5 (US3) Retro — Argument substitution

**Slices**: US3.a (argument stage + four patterns), US3.b (ARGUMENTS append-fallback), US3.c (end-to-end + reviewer pass + closeout)
**Phase 4 baseline**: PR #118 merged @ 914cb0f. Test total 1089/141/16.

## What this phase will land

Wire the F3 substitution-engine's Stage 3 (argument substitution) + Stage 4 (ARGUMENTS append fallback). Lights up the final two stages of the 4-stage pipeline. After this US, the full substitution layer is operational.

- Stage 3: regex `\$ARGUMENTS\[(\d+)\]|\$ARGUMENTS|\$(\d+)|\$([a-z_][a-z0-9_]*)` substitutes 4 argument patterns
- Stage 4: `ARGUMENTS:` append-fallback footer when caller supplied args but stage 3 didn't replace anything
- Caller-arg coercion: `Single(s)` shell-split vs `Object{}` named lookup per the 6-row coercion table
- `prompt_argument_mismatch` error path: caller supplies named keys not in declaration set

## Critical inherited invariant

Per US2.d's single-sweep refactor: Stage 1 + Stage 2 are merged into one regex (`COMBINED_RE`). Stage 3 must EITHER:
- Be merged into the same sweep (cleanest — extend COMBINED_RE with the four argument patterns), OR
- Run as a separate sweep AFTER Stage 1+2 but with explicit tests verifying that Stage 1+2 substituted values containing `$ARGUMENTS` / `$N` / `$name` text are NOT re-substituted by Stage 3.

The structurally-correct approach is the merge. If the merge is too invasive, the second approach is acceptable with rigorous testing.

## Workarounds

### Option A (structural fix) over Option B (separate sweep)
The brief offered two approaches to integrating Stage 3:
- Option A: extend the union regex to include Stage 3 patterns (6 capture groups)
- Option B: separate Stage 3 sweep after the Stage 1+2 sweep

Picked Option A. The rationale: Option B would re-introduce the no-rescan invariant violation that US2.d had just fixed (a hostile plugin could set `plugin_name = "$0"` and `${TOME_PLUGIN_NAME}` would resolve to `$0` text that Stage 3 would then substitute). Single-sweep beats two-sweep when no-rescan is in spec.

Result: the structural fix from US2.d carries through cleanly to US3. The 6-capture-group dispatch in `render()` resolves all 3 stages in one pass.

### `apply_arguments` F3 stub replaced (consistent pattern)
F3's separate-sweep stub was replaced by the single-sweep design. The `ARGUMENTS_RE` slot in `regex_sets.rs` was removed (subsumed by `combined_regex`). Same cleanup as US2.d removed `apply_builtins` and `apply_env` as separate stages.

### 0-indexed positional convention surprise
`$0` is the first positional, matching the contract verbatim and Claude Code's `$ARGUMENTS[0]` convention. Plugin authors familiar with shell's `$1` (1-indexed) will need a moment to adjust. Worth flagging in operator docs at v0.5.0 release.

### Test fixture `stage_workspace_with` now at 4 file-local variants
Across `mcp_prompts.rs`, `mcp_prompts_list_json_shape.rs`, `mcp_prompts_get_json_shape.rs`, `substitution_pipeline.rs`, and now `entry_e2e.rs` (the latter uses a variant matching `substitution_pipeline.rs`'s shape). They carry materially different fixture knobs (config persisted to disk vs not, env guards, plugin/workspace data-dir guards). Decision: defer promotion until a fifth consumer lands with materially-similar shape, then fold into a builder-style helper. Documented in `entry_e2e.rs` module preamble.

### Reviewer pass T-M1 test originally failed (Stage 4 footer)
The first attempt at `stage_2_substituted_value_containing_dollar_pattern_is_not_rescanned_by_stage_3` asserted just the body without considering Stage 4 append-fallback. Since the test passes args but the body has no Stage 3 references, Stage 4 fires and appends `\n\nARGUMENTS: <value>`. Fixed by including the expected footer in the assertion — the key invariant (the `$0` from env value NOT being substituted) is still verified by the body containing literal `$0`.

### R-M2 deletion left orphan doc comment
Deleting `_entry_identity_from_record` (R-M2) left the function's doc comment block hanging above the next item, which compiled fine but produced an error from the next item's existing doc. Had to scrub the orphan comment block. Lesson: when deleting a doc-commented item, scrub the comment block in the same edit. Also: when deleting a function, check for unused imports it may have brought in (had to drop `SkillRecord` import too).

## Patterns established

### 4-reviewer parallel-pass continues to pay
US3's reviewer pass surfaced 4 actionable majors that the implementation sub-agents missed (M1 substring check; M2 dead code; M3 clone micro-opt; M4 empty-Object edge case) plus 1 test gap. The pattern is now 3-for-3 across Phase 5 (US1.d, US2.d, US3.d). Cost: ~5 min wall clock for 4 agents in parallel + 10-15 min for orchestrator to consolidate + apply fixes. Compared to shipping the major-tier issues for users to find: invaluable.

### Structural fixes inherited automatically
US2.d's single-sweep refactor required no re-work in US3 — extending the union regex with 4 more patterns + 3 more capture groups was a mechanical extension. The structural invariant (no rescanning) carried through to Stage 3 automatically. Lesson: structural fixes pay compounding dividends; targeted fixes have to be re-litigated at every subsequent slice.

### `body_has_bare_arguments` as public helper from substitution layer
The R-M1 fix needed `mcp::prompts` to call substitution-internal regex machinery. Rather than promote private constants (`pub(super)` → `pub(crate)`), added a public helper function on the substitution module that encapsulates the regex inspection. Keeps the regex internals encapsulated while exposing a high-level capability to consumers.

### Inline fixes when reviewer pass is small
US3.d had 0 BLOCKERS + small majors. Orchestrator applied the fixes directly without dispatching a sub-agent. Threshold: if total work fits in <5 file edits + <30 LOC change, faster to do inline than dispatch sub-agent (round-trip cost ~3-5 min).

### `tests/entry_e2e.rs` validates the full surface end-to-end
First end-to-end test exercising the complete enable → search → get → prompts pipeline. Catches integration bugs that slice-level tests can't see. Pattern: every US closeout should have at least one e2e test validating the full surface added in that US.
