## What this phase will land

Wire the F3 substitution-engine's Stage 3 (argument substitution) + Stage 4 (ARGUMENTS append fallback). Lights up the final two stages of the 4-stage pipeline. After this US, the full substitution layer is operational.

- Stage 3: regex `\$ARGUMENTS\[(\d+)\]|\$ARGUMENTS|\$(\d+)|\$([a-z_][a-z0-9_]*)` substitutes 4 argument patterns
- Stage 4: `ARGUMENTS:` append-fallback footer when caller supplied args but stage 3 didn't replace anything
- Caller-arg coercion: `Single(s)` shell-split vs `Object{}` named lookup per the 6-row coercion table
- `prompt_argument_mismatch` error path: caller supplies named keys not in declaration set

## Critical inherited invariant

Per US2.d's single-sweep refactor: Stage 1 + Stage 2 are merged into one regex (`COMBINED_RE`). Stage 3 must EITHER:
- Be merged into the same sweep (cleanest — extend COMBINED_RE with the four argument patterns), OR
- Run as a separate sweep AFTER Stage 1+2 but with explicit tests verifying that Stage 1+2 substituted values containing `$ARGUMENTS` / `$N` / `$name` text are NOT re-substituted by Stage 3.

The structurally-correct approach is the merge. If the merge is too invasive, the second approach is acceptable with rigorous testing.
