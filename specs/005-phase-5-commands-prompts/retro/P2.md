## Workarounds

### F2 reduced to no-op
The original plan said "promote `regex` from transitive to direct". On
inspection `regex = "1"` was already a direct dep since Phase 1 — used by
`catalog::git::scrub_credentials`. F2 reduced to a binary-size record in
`RELEASE-BINARY-SIZE.md` and a one-line `chore(deps)` commit. The Phase 5
plan's research §R-2 wording ("promoted from transitive to direct") was
based on a stale read of the workspace tree.

### Contract amendment: `regex_sets.rs` not `regex.rs`
Task T027 names the regex-cache file `src/substitution/regex.rs`. Using
that name shadows the `regex` crate inside the substitution module and
forces clumsy `::regex::Regex` paths at every use site. F3 names the file
`regex_sets.rs` instead. Tasks.md T027 amended in-line.

### Sub-agent didn't auto-commit
The first F3 sub-agent dispatch produced clean files but reported "test
still running" without finishing the verify+commit step. The orchestrator
verified locally (`cargo fmt`/`clippy`/`test --test substitution_skeleton`
all green) and committed. **Lesson for next slice**: prefer running `cargo
test` in the orchestrator after sub-agent return rather than relying on
the sub-agent's self-verification — round-trip cost is the same and the
orchestrator's commit history stays clean.

## Patterns established

### Pre-allocation slice with `#[allow(dead_code)]` markers
F3 ships 7 new files where only `mod.rs::render()` is reachable from F3
itself. Internal helpers (`apply_builtins`/`apply_env`/`apply_arguments`/
`ensure_plugin_data`/`ensure_workspace_data` + the 3 regex `OnceLock`
slots) carry `#[allow(dead_code)]` to silence clippy until US1+US2+US3
wire them. Convention: every `#[allow(dead_code)]` in F3 should have a
short doc-comment line saying which US lights it up. When the consumer
slice adds the production logic, the `#[allow]` comes off in the same PR.

### Inline contract amendments on collision
F1 caught a hard collision (Phase 5 contract proposed codes 21/22/23,
already held by Phase 2's plugin lifecycle). Same shape as Phase 4 F3's
SummariserFailure-vs-PluginNotFound collision. Pattern: pick a clean
free cluster, fix the contract docs in the same commit as the code
implementation, document the amendment in the commit body + retro. Phase
5's cluster ended up at 25–29 (clean and contiguous).

### Doc-fix breadth on a contract amendment
The exit-code fix touched 11 files because the original codes (21/22/23)
were referenced across contracts/, data-model.md, quickstart.md,
research.md, tasks.md, and PRDs/phase-5.md. Lesson: when amending a code
in a closed-enum contract, grep ALL planning artefacts for the bare
number — not just the contract file. The `[a-z_]*_code [0-9]+`
pattern occasionally hides the reference (e.g. in tables and slug maps).

### Pre-commit hook covers fmt + typos + clippy + cog atomically
The pre-commit hook ran each of the four checks before each commit. Every
single F1/F2/F3 commit passed first-time without bypass — the
discipline of running `cargo fmt && cargo clippy -- -D warnings && cargo
test --test <new>` before `git commit` continues to be cheap and
worthwhile.

## Decisions deferred

### Mapper refresh deferred from Phase 2 close
Tasks T035 schedules `/sdd:map incremental` at Phase 2 close. The
production change in F1+F2+F3 is small (5 enum variants + a stub module);
the codebase docs in `.sdd/codebase/` were refreshed at v0.4.0 close and
don't have stale structural facts. **Decision**: defer the mapper refresh
to US1 close (or US2 close at latest), where it'll capture both the
skeleton's structure AND the first production wiring of the substitution
module. This saves ~15 min of mapper time on a small slice.

The Phase 4 P8 retro recommended mid-phase mapper rotation for Phase 5+,
which this defers slightly but stays in spirit.
