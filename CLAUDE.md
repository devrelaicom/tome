## Active Technologies

- **Language**: Rust stable, MSRV pinned at `rust-version = "1.93"` (verified in CI).
- **CLI**: `clap` (derive feature) ‚Äî `--help` / `--version` / global flags.
- **Config / manifest**: `serde` + `serde_derive`, `toml` ‚Äî Tome-owned structs use `#[serde(deny_unknown_fields)]`.

- **Paths**: `directories` (XDG-aware).
- **Hashing**: `sha2`, `hex` (cache directory naming, model checksums, content hashes).
- **Atomic writes**: `tempfile` (registry, catalog cache, models dir atomicity).
- **Signal handling**: `ctrlc` (SIGINT cancellation; exits with code 8).
- **Colour / NO_COLOR**: `anstream` + `anstyle` (transitive via clap 4).
- **Regex**: `regex` (credential scrubbing in `src/catalog/git.rs`).
- **Time**: `time` (RFC 3339 timestamps).
- **Semver**: `semver`.

### Phase 2 additions

- **Embedded database**: `rusqlite` with the `bundled` feature ‚Äî statically linked SQLite, no system dep.
- **Vector search**: `sqlite-vec` C extension vendored under `vendor/sqlite-vec/`, compiled in via `build.rs`.
- **Inference**: `fastembed-rs` wrapping `ort` (ONNX Runtime). CPU execution provider only; CUDA / CoreML / DirectML disabled.
- **Models** (downloaded at runtime; not bundled): `bge-small-en-v1.5` INT8 (~45 MB, MIT), `bge-reranker-base` INT8 (~280 MB, MIT). Stored under `${XDG_DATA_HOME}/tome/models/`.
- **Progress / spinners**: `indicatif`.
- **Tables**: `comfy-table`.
- **Colours**: `owo-colors` (native `NO_COLOR`).
- **Prompts**: `inquire` (Select / MultiSelect / Confirm; refuses on non-TTY).
- **HTTP**: `reqwest` with `blocking` + `rustls-tls`, `default-features = false`.

**Strictness boundary** (FR-013a): `#[serde(deny_unknown_fields)]` applies to Tome-owned inputs (config, model `manifest.json`, index `meta` rows). Third-party inputs (`plugin.json`, `SKILL.md` YAML frontmatter) parse leniently ‚Äî forward-compat with upstream additions.

**Not used**: `tokio`, `libgit2`/`git2`, `atty`, `colored`, `lazy_static`, `once_cell` (std `OnceLock` covers it). Phase 2 stays synchronous.

## Architectural Constraints (from the constitution)

- **Sync only.** No async runtime. `reqwest::blocking`, `rusqlite`, and `fastembed-rs` are all sync. The MCP server is the future forcing function for async.
- **Inherit `git`.** Shell out to system `git`. Never vendor a Git library.
- **Closed error set.** `TomeError` has no `Other`/`Unknown` arm. Every Phase 2 failure class has its own enumerated variant and exit code (see `specs/002-phase-2-plugins-index/contracts/exit-codes.md`).
- **Strictness boundary.** Tome-owned declarative inputs are strict (`#[serde(deny_unknown_fields)]`); third-party inputs (`plugin.json`, SKILL.md frontmatter) are lenient ‚Äî see spec FR-013a. The strict-on-Tome-owned principle is enforced by `tests/manifest_strictness.rs`.
- **Atomic writes.** Registry, cache, models directory, and index DB mutations are atomic. SQLite WAL + a Tome-owned advisory lockfile (`index.lock`) provide the index concurrency contract (FR-040).
- **Credential scrubbing at the boundary.** `git::scrub_credentials` extends to model download URLs and `reqwest` error chains.
- **50 MB binary cap** (revised from 10 MB on 2026-05-13 ‚Äî see `retro/P3.md` ¬ß"Binary-size cap revision"). CI asserts `stat -c%s target/release/tome` on Linux. `ort` (CPU-only static) is the load-bearing dep; profile is `lto = "thin"`, `panic = "abort"`, `strip = "symbols"`. If breached, the plan revises ‚Äî the discipline is non-waivable (NFR-001); the specific number is sized to current reality plus headroom.
- **Licence allowlist.** Unchanged. Every Phase 2 dep verified inside the allowlist. `cargo-deny` enforces. Downloaded models (BGE family, MIT) are surfaced in `tome models list`.

## Conventions

- **Commits**: Conventional Commits. Enforced locally by `cocogitto` (`cog verify`) in the `commit-msg` hook (versioned under `.githooks/`). Format: `type(scope): subject`.
- **Branching**: trunk-based; short-lived branches off `main`.
- **PRs**: small batches ‚Äî ~400 lines or 2 modules max as a soft cap.
- **Comments**: explain *why*, not *what*. Reader knows Rust.
- **Modules**: capability-organised. Phase 1: `catalog`, `config`, `paths`, `error`, `output`, `logging`. Phase 2 adds: `plugin` (manifest/frontmatter/lifecycle), `index` (db/schema/migrations/vec-ext/skills/query/meta/integrity/lock), `embedding` (fastembed wrapper + stub + registry + download + runtime), `presentation` (wraps comfy-table / indicatif / owo-colors / inquire). Phase 3 adds: `workspace` (scope / resolution / inventory) and `mcp` (mod / runtime / log / preflight / server / state / tools ‚Äî the one async island, structurally enforced by `tests/sync_boundary.rs`).
- **Errors**: `thiserror` inside modules; `anyhow` at the application boundary.
- **Field-method coexistence** (Phase 3 / F1): When a refactor would rename a public struct field across many call sites, add accessor methods first (e.g. `Paths::config_file_for(&Scope)` alongside the existing `Paths.config_file` field). The accessor's `_for(&Scope)` suffix often communicates intent better than the renamed field would have; the rename can then be deferred or shelved.
- **Specific-over-generic exit codes** (Phase 3 / F8 preflight): When a failure matches an existing Phase 1/2 variant, emit that code rather than a generic residual (`ModelMissing` 30 / `ModelChecksumMismatch` 32 / `EmbedderNameDrift` 41 / `SchemaVersionTooNew` 73 all win over `McpStartupFailed` 60). The closed `TomeError` set keeps every alternative discoverable.
- **Test injection via `#[doc(hidden)] pub static`** (Phase 3 / F7): Integration tests under `tests/` consume the library without `cfg(test)` visibility, so `#[cfg(test)] thread_local!` injection points are unreachable from there. Gate with `#[doc(hidden)] pub static` instead; document the test-only intent in the doc comment. Phase 7's `tests/schema_migration_e2e.rs` is the first consumer.
- **Silent compute / emit wrapper pattern** (Phase 3 / US1.b): When a CLI command's compute path is reused by a non-CLI surface (MCP, library API, RPC), split into a `pipeline(args, deps) -> Result<Outcome, Error>` (no I/O side-effects) plus a `run_with_deps(args, deps, mode) -> Result<Outcome, Error>` that calls `pipeline` then emits per `mode`. Tests + alternative callers use `pipeline`; the CLI uses `run_with_deps`. First applied splitting `query::run_with_deps` so `mcp::tools::search_skills` could reuse the KNN+rerank logic without the stdout/stderr emit step (FR-221 ‚Äî protocol channel is sacred).
- **`spawn_blocking` for sync work inside async handlers** (Phase 3 / US1): MCP tool handlers are `async fn` (required by rmcp) but their actual work (rusqlite queries, fastembed inference, fs::read) is sync. Pattern: do cheap async-friendly work directly (validation, config load), then `tokio::task::spawn_blocking(move || { ... sync work ... }).await`. The single-threaded runtime stays responsive; inference latency doesn't hold up the reactor.
- **Atomic populated-directory landing** (Phase 3 / US2.b): when a multi-file directory must appear either complete or not at all to readers, build it in a sibling staging dir on the same filesystem and rename once. Pattern: `tempfile::Builder::new().prefix(".tome.tmp.").tempdir_in(target_parent)?` ‚Üí populate ‚Üí `staging.keep()` (drops the auto-cleanup guard, returns `PathBuf`) ‚Üí `std::fs::rename(staged, target)`. `std::fs::rename` is POSIX-atomic when source and target share a filesystem; the staging-inside-target-parent invariant makes that true. Distinct from `NamedTempFile::persist` (file-level). For replace-existing semantics, rename the existing target aside to `.<name>.old/` first, then rollback on staging-rename failure. Used in `src/workspace/init.rs`; `tempfile::TempDir::persist` doesn't exist (the contract referenced it; we use `keep()` + `std::fs::rename` instead).
- **Opt-in registry file** (Phase 3 / US2.b): for "track this for me, but only if I ask" features, write a helper like `inventory::append_if_registry_exists(path, item)` that no-ops when the registry file is absent. The user touches the file once to opt in; subsequent operations append. Dedupe by exact-string match against the canonicalised item. Pattern lives in `src/workspace/inventory.rs`; reusable for any future opt-in tracking surface.
- **Emit-only `Serialize` records** (Phase 3 / US2): `WorkspaceInfo`, `InitOutcome`, and similar wire-shape types are never deserialised ‚Äî they exist to render `--json` for stdout. No `#[serde(deny_unknown_fields)]` because the strictness boundary only applies to inputs. Pin the wire format with a byte-stable JSON serialisation test instead (e.g. `tests/workspace_info.rs::info_json_shape_is_byte_stable_for_bootstrap_not_yet`).
- **Reuse existing closed-set variants over promoting new ones** (Phase 3 / US2.b): when a new failure mode maps semantically onto an existing `TomeError` variant + exit code, prefer the reuse over a dedicated variant. Cost: a slightly off Display message (e.g. `CatalogAlreadyExists("workspace at /path/.tome")` displays as "catalog `workspace at /path/.tome` is already registered"). Benefit: zero variants added, zero exit-code-table churn, zero JSON envelope shape change. Promote to a dedicated variant only if a specific failure mode emerges that needs to be distinguished from the existing one. The contract explicitly green-lit this reuse for workspace-already-initialised ‚Üí `CatalogAlreadyExists` (4).
- **Content-addressed shared resource with reference counting** (Phase 3 / US3.b): when an on-disk resource is shared across scopes (e.g. the `catalogs/<sha256>/` cache directories), don't unconditionally delete on remove. Build an ad-hoc reference table by enumerating every scope's config that could reference the URL/key, and only delete when the count reaches zero. Implementation lives in `catalog::store::reference_count(url, paths) -> Vec<Scope>`. The reference enumeration is unlocked ‚Äî concurrent removes race benignly (one wins, the other no-ops); a remove + add race may leave a dangling reference recoverable by re-fetching. Same TOCTOU profile as Phase 2's `cascade_disable_for_catalog` pre-check. Reusable for any future shared-on-disk resource.
- **`reuse_existing: bool` rollback gating** (Phase 3 / US3.b): when a function may either CREATE a resource or REUSE an existing one (e.g. `catalog::add` ‚Äî clone or refcount-reuse), name the boolean after the path taken ("did we take the reuse path?") rather than after the rollback action ("should we delete on save failure?"). The save-failure rollback then becomes `if !reuse_existing { remove_dir_all(...) }` ‚Äî read-time intent matches the question being asked.
- **F1's mechanical sweep paid off in US3** (Phase 3 / US3.a): when a Foundational slice does a wholesale mechanical sweep (in this case threading `ResolvedScope` and `Paths::*_for(&scope)` through every command), the downstream user-story slices that "use" the sweep often reduce to "prove it works end-to-end" ‚Äî i.e. the integration test IS the deliverable, with no production code changes. Cost: zero. Benefit: the verification commit doubles as the regression net.
- **Helper-visibility promotion to share state across surfaces** (Phase 3 / US4): when two surfaces must report the same value for the same subsystem (e.g. `tome status` + `tome doctor` on models / index / drift), promote the shared helper from private to `pub` rather than duplicating compute. Three-character edit, zero risk of the two surfaces disagreeing. Documented as "read-only diagnostic helpers" so future callers know the intent. Applied to `commands::status::check_model` / `check_index` / `check_drift`.
- **`home: &Path` as test-isolation hook** (Phase 3 / US4): when a library function would otherwise read `$HOME` directly, accept it as a parameter and have the CLI wrapper resolve env. Tests pass a `TempDir`-rooted path; production passes the real `$HOME`. Drops the dependency on `serial_test` for env-isolated tests. Applied to `doctor::assemble_report(scope, paths, home, verify)` for harness-probe testability; same idiom applies to any future read-from-env operation.
- **`subsystem` string routing for tagged repair lists** (Phase 3 / US4): when a list of items needs per-item dispatch (here: `SuggestedFix.subsystem` values like `"embedder"` / `"reranker"` / `"catalog:<name>"` / `"schema"`), encode the dispatch key in a string field rather than promoting to an enum. `.strip_prefix("catalog:")` extracts per-instance data. Flat if-else dispatch ladder, wire-friendly. Trade-off: less type safety than an enum; acceptable up to ~6 arms.
- **`re_assemble` post-mutation pattern** (Phase 3 / US4): when an "apply" function mutates per-field state in place (e.g. `fixes::apply_one` updates `report.embedder` / `report.catalogs` after each repair), expose a sibling `re_assemble` that recomputes derived state (`suggested_fixes`, `overall`) from those fields without re-running expensive checks. Halves the FS cost on the catalog-enumeration + harness-probe sides.
- **RAII `MigrationsGuard` for thread-local injection** (Phase 3 / US5): `MIGRATIONS_OVERRIDE` is a `thread_local!` per F7 convention. Tests install it via a 1-line `MigrationsGuard::install(&[...])` whose `Drop` impl clears the slot ‚Äî survives panics, no manual teardown, no cross-test leakage. The migration step itself is a named `fn` item (not a closure) because `Migration.apply` is `fn(&Transaction) -> Result<(), TomeError>`. Reusable shape for any future test-only `thread_local!` injection point: pair the `pub static` with a guard struct in the consumer test file. Used in `tests/schema_migration_e2e.rs`, `tests/atomicity.rs`, `tests/doctor.rs`.
- **Generate-at-setup fixture DBs over committed binaries** (Phase 3 / US5): when a test needs an "older-schema" or "newer-schema" SQLite file, write a helper like `write_index_db_with_schema_version(path, version)` that bootstraps a minimal `meta`-only DB at the requested version. Beats committing `tests/fixtures/older-schema.db` binaries: no binary churn in PR diffs, no opaque blobs in git history, no surprise when the production schema layout drifts. The fixture intentionally has *only* `meta` so synthetic migrations can create whatever tables they want without conflicting with the production schema.
- **SIGINT mid-tx modelled as deliberate `Err`** (Phase 3 / US5): the `catalog::git::CANCELLED` static cannot be used inside test bodies because cargo runs tests in the same process ‚Äî flipping it races every other test in the binary (same discipline as `atomicity_enable.rs`). For "interrupted mid-transaction" tests, return a deliberate `Err` from the migration closure: the per-step transaction rollback is identical regardless of whether the abort came from a signal handler or a closure-level failure, so the assertion targets (table absent, schema_version unchanged) are the same. Generalisable to any test that wants to verify rollback behaviour without process-global signal manipulation.
- **Synthetic SuggestedFix injection for dispatch-path testing** (Phase 3 / US5): when a framework SHIPS the dispatch path for a class of fixes but DOESN'T YET produce them (Phase 3 ships zero registered migrations, so `build_suggested_fixes` never emits `subsystem: "schema"`), tests can manually push a synthetic `SuggestedFix { auto_fixable: true, ... }` into the assembled report before calling `doctor::fixes::apply`. The dispatch happens through the real production code; only the *trigger* is synthetic. Reusable shape for any framework whose production-side trigger logic ships separately from its dispatch + execute logic. Applied in `tests/doctor.rs::fix_runs_forward_schema_migration_end_to_end`.
- **Framework ships empty, e2e tests live on synthetic fixtures** (Phase 3 / US5): the schema-migration framework ships with `pub const MIGRATIONS: &[Migration] = &[]` ‚Äî zero registered migrations. The Phase 3 e2e tests don't add real migrations; they exercise the framework via `MIGRATIONS_OVERRIDE` injection so Phase 4+ migrations land on tested rails. Pattern: when shipping a framework whose first production consumer is a future phase, ship the test harness (override hook + e2e tests against synthetic registrations) NOW so the rails are battle-tested before any real migration runs against a user's DB.
